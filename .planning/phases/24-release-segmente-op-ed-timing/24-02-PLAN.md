Neue Datei `useReleaseSegments.ts` im Episode-Version-Edit-Ordner erstellen.

Der Hook liest anime_id, group_id und version aus dem EpisodeVersionEditorContext (wird als Parameter uebergeben, nicht selbst geladen) und kapselt alle Segment-CRUD-Operationen.

```typescript
'use client'

import { useCallback, useEffect, useState } from 'react'

import {
  getAnimeSegments,
  createAnimeSegment,
  updateAnimeSegment,
  deleteAnimeSegment,
  getAdminAnimeThemes,
} from '@/lib/api'
import type { AdminThemeSegment, AdminThemeSegmentCreateRequest, AdminThemeSegmentPatchRequest } from '@/types/admin'
import type { AdminAnimeTheme } from '@/types/admin'
import { getRuntimeAuthToken } from '@/lib/api'

interface UseReleaseSegmentsOptions {
  animeId: number | null
  groupId: number | null
  version: string | null
}

export function useReleaseSegments({ animeId, groupId, version }: UseReleaseSegmentsOptions) {
  const [authToken] = useState(() => getRuntimeAuthToken())
  const [segments, setSegments] = useState<AdminThemeSegment[]>([])
  const [themes, setThemes] = useState<AdminAnimeTheme[]>([])
  const [isLoading, setIsLoading] = useState(false)
  const [errorMessage, setErrorMessage] = useState<string | null>(null)

  const load = useCallback(async () => {
    if (!animeId || !authToken) return
    setIsLoading(true)
    setErrorMessage(null)
    try {
      const [segRes, themeRes] = await Promise.all([
        getAnimeSegments(animeId, groupId, version, authToken),
        getAdminAnimeThemes(animeId, authToken),
      ])
      setSegments(segRes.data)
      setThemes(themeRes.data)
    } catch (error) {
      setErrorMessage(error instanceof Error ? error.message : 'Fehler beim Laden der Segmente.')
    } finally {
      setIsLoading(false)
    }
  }, [animeId, groupId, version, authToken])

  useEffect(() => { void load() }, [load])

  async function create(input: AdminThemeSegmentCreateRequest): Promise<boolean> {
    if (!animeId || !authToken) return false
    try {
      const res = await createAnimeSegment(animeId, input, authToken)
      setSegments((current) => [...current, res.data])
      return true
    } catch (error) {
      setErrorMessage(error instanceof Error ? error.message : 'Segment konnte nicht angelegt werden.')
      return false
    }
  }

  async function update(segmentId: number, input: AdminThemeSegmentPatchRequest): Promise<boolean> {
    if (!animeId || !authToken) return false
    try {
      await updateAnimeSegment(animeId, segmentId, input, authToken)
      await load()
      return true
    } catch (error) {
      setErrorMessage(error instanceof Error ? error.message : 'Segment konnte nicht aktualisiert werden.')
      return false
    }
  }

  async function remove(segmentId: number): Promise<boolean> {
    if (!animeId || !authToken) return false
    try {
      await deleteAnimeSegment(animeId, segmentId, authToken)
      setSegments((current) => current.filter((s) => s.id !== segmentId))
      return true
    } catch (error) {
      setErrorMessage(error instanceof Error ? error.message : 'Segment konnte nicht geloescht werden.')
      return false
    }
  }

  return { segments, themes, isLoading, errorMessage, create, update, remove, reload: load }
}
```

**`SegmenteTab.tsx` erstellen:**

Komponente empfaengt Props: `{ animeId: number | null, groupId: number | null, version: string | null }`.

Nutzt `useReleaseSegments({ animeId, groupId, version })`.

**Tabellen-Bereich:**

Spalten: Typ (Badge), Name, Episoden, Zeitbereich, Quelle, Aktionen.

Typ-Badge: Inline-Badge mit kurzem Code und Farbe:
- Name enthaelt "OP" → gruener Badge mit Text "OP"
- Name enthaelt "ED" → violetter Badge mit Text "ED"
- Name enthaelt "Insert" oder "IN" → oranger Badge mit Text "IN"
- Name enthaelt "Outro" oder "PV" → grauer Badge mit Text "PV"
- Sonstige → blauer Badge mit theme_type_name

Episoden-Spalte: `${start_episode ?? '?'} – ${end_episode ?? '?'}`
Zeitbereich-Spalte: `${start_time ?? '--'} → ${end_time ?? '--'}`
Quelle-Spalte: source_jellyfin_item_id gekuerzt auf max. 20 Zeichen + "..." wenn laenger, oder "–"

Aktions-Buttons: Bleistift-Icon (lucide-react `Pencil`, Groesse 14) und Papierkorb-Icon (`Trash2`, Groesse 14). Loeschen zeigt `window.confirm("Segment wirklich loeschen?")`.

**Seitenleisten-Formular (rechts, slides in bei panelOpen=true):**

Felder:
- Typ-Dropdown: `<select>` mit Optionen aus `themes` (theme_type_name als Label, theme.id als value)
- Name (optional): `<input type="text" placeholder="optional">`
- Episodenbereich: zwei `<input type="number" min="1">` — "von" und "bis"
- Zeitbereich Start: `<input type="text" placeholder="00:00:00">` fuer HH:MM:SS
- Zeitbereich Ende: `<input type="text" placeholder="00:00:00">` fuer HH:MM:SS
- Jellyfin Item ID (optional): `<input type="text">`

Formular-State als lokales useState mit Feldern:
`{ themeId: '', name: '', startEpisode: '', endEpisode: '', startTime: '', endTime: '', sourceJellyfinItemId: '' }`

Beim Oeffnen des Bearbeiten-Dialogs: State mit den Werten des bestehenden Segments vorbelegen.
Beim Speichern:
- Neu: `create({ theme_id: parseInt(themeId), version: version || 'v1', fansub_group_id: groupId, ... })`
- Bearbeiten: `update(segment.id, { ... })`
- Bei Erfolg: Panel schliessen, State zuruecksetzen

**Timeline-Visualisierung am unteren Ende des Tabs:**

Horizontaler Container. Fuer jedes Segment das sowohl start_time als auch end_time hat:
- Konvertiere HH:MM:SS in Sekunden: `parseTimeToSeconds(t: string): number`
- maxEnd = Maximum aller end_time-Sekunden (Fallback: 300 wenn leer)
- Pro Segment: farbiger Block mit `left: (start/maxEnd * 100)%`, `width: ((end-start)/maxEnd * 100)%`
- Mindestbreite 2%, Farbe gemaess Typ wie Badges
- Tooltip (title-Attribut): `${theme_type_name}: ${start_time} – ${end_time}`
- Label "Timeline" ueber der Leiste

Wenn keine Segmente mit Zeitbereich: Hinweistext "Keine Zeitbereiche fuer Timeline verfuegbar."
