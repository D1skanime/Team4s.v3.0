# Phase 04-02 Plan - Edit Route Provenance UX And Resync Preview

Status: planned

## Objective

Integrate persisted provenance and explicit metadata resync into the anime edit route so operators can understand source linkage and apply safe refreshes without leaving the editor.

## Requirements

- `JFIN-03` Persisted anime refresh is accessible from the edit route
- `OWNR-01` Ownership/provenance state is understandable in UI
- `OWNR-05` Resync is preview-first and explicit
- `RLY-02` Errors and blocked actions are clearly communicated

## Implementation Scope

### Edit Route UI

- Add a provenance panel or section to `/admin/anime/[id]/edit`.
- Show linked/manual state with clearer semantics than the current binary ownership badge.
- Display source details in operator-readable form:
  - source type
  - folder/path identity
  - linked status

### Resync UX

- Add preview action before apply.
- Show field-level diff rows and skipped/manual-protected fields.
- Provide German feedback for:
  - no linkage
  - preview failure
  - apply success
  - apply partial/no-op

### Reuse Strategy

- Reuse the existing `JellyfinSyncPanel` preview/apply interaction language where it fits.
- Do not merge episode sync and metadata resync into one overloaded surface.

## Non-Goals

- redesign of the full edit route layout
- public provenance display
- AniSearch provenance

## Verification

- component tests cover panel states and diff rendering
- edit route remains reachable and stable after rebuild
- linked and manual anime both present correct affordances
