# Phase 04-01 Plan - Persisted Provenance And Metadata Resync Contracts

Status: planned

## Objective

Add the backend and typed contract layer required to inspect persisted Jellyfin linkage and preview/apply safe metadata refreshes for an existing anime record.

## Requirements

- `JFIN-03` Persisted anime can be refreshed from linked Jellyfin metadata
- `OWNR-02` Persisted linked source identity is inspectable
- `OWNR-03` Fill-only metadata refresh protects manual values
- `OWNR-04` Explicitly cleared values can be refilled via operator-triggered refresh
- `OWNR-05` Operator can preview linked-source changes before apply
- `RLY-02` Failure states are explicit and safe

## Implementation Scope

### Backend

- Expose an admin edit-context or provenance endpoint for persisted anime.
- Return linked-source metadata needed by the editor:
  - linkage state
  - source label/value
  - folder/path identity
  - provider item identity when available
- Add preview/apply endpoints or endpoint modes for persisted metadata resync.
- Keep apply behavior fill-only by default and explicit about skipped fields.
- Return stable, operator-readable diff payloads for the frontend.

### Frontend Types

- Extend admin type definitions for persisted provenance and resync preview/apply payloads.
- Avoid leaking admin-only provenance concerns into unrelated public models unless runtime requires it.

## Non-Goals

- episode sync changes
- bulk anime resync
- asset binary upload mechanics

## Verification

- backend tests cover provenance lookup and preview/apply semantics
- frontend type usage compiles cleanly
- runtime requests return predictable error shapes for unlinked anime
