**SCOPE-KRITERIUM — was ist user-facing in Go:**
Ein Go-String ist user-facing wenn er in einer dieser Konstruktionen vorkommt:
- `badRequest(c, "HIER")`
- `gin.H{"message": "HIER"}` oder `gin.H{"error": "HIER"}`
- `gin.H{"details": "HIER"}` (nur wenn die Details im Frontend angezeigt werden)
- `writeInternalErrorResponse(c, "HIER", ...)` — nur das erste String-Argument
- `c.JSON(statusCode, gin.H{...})` — alle String-Werte die an Frontend-Keys gehen
- Error-Meldungen die direkt in Response-Body landen (nicht nur im error-Objekt
  das in Logs landet)

**NICHT aendern:**
- `fmt.Errorf("...")` die nur als interner Error-Context dienen (zweites/drittes
  Argument in writeInternalErrorResponse)
- `log.Printf("...")`
- Kommentare
- Variablen- oder Funktionsnamen
- SQL-Strings
- Datei-Pfad-Konstruktionen

**ERSETZUNGSTABELLE fuer Go-Strings (vollstaendig):**

```
ungueltige    → ungültige
ungueltiger   → ungültiger
ungueltiges   → ungültiges
ungueltigen   → ungültigen
ungueltig     → ungültig
verfuegbar    → verfügbar
nicht verfuegbar → nicht verfügbar
fuer          → für        (NUR in user-facing message strings)
waehlen       → wählen
auswaehlen    → auswählen
Loeschen      → Löschen

oeffnen       → öffnen

Aenderungen   → Änderungen

voruebergehend → vorübergehend

```
