## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Eyebrow-to-title gaps, inline helper spacing, summary list gaps |
| sm | 8px | Status pill gaps, checkbox gaps, compact field meta spacing |
| md | 16px | Default card gap, card stack spacing, mobile card padding floor |
| lg | 24px | Section padding, desktop page gutters, create workspace separation |
| xl | 32px | Major panel separation and page-level breathing room |
| 2xl | 48px | Page bottom spacing and full-width section breaks |
| 3xl | 64px | Reserved for future page hero spacing; do not introduce in this phase unless a new page-level section is added |

Exceptions: `12px` may continue for eyebrow text spacing and compact helper rhythm because the live token set already includes `--space-3: 12px`; all new Phase 12 layout changes should otherwise use the declared 4/8/16/24/32/48/64 scale. Interactive buttons and input controls must keep a minimum 44px height; the existing title input may remain 48px.

## Phase 12 Interaction Contract

### Surface placement

- Place the AniSearch create card directly above the existing `Jellyfin suchen` action seam inside the `ManualCreateWorkspace` title action area.
- Keep AniSearch and Jellyfin in the same source-action cluster so operators can compare sources without leaving the create form.
- Preserve the current create route structure, card styling, border radius, and shadow language from the existing create workspace and results panels.

### Source hierarchy

- AniSearch is the visually primary source action for this phase.
- Jellyfin remains available, but it must render as a secondary action with neutral styling.
- Show a short helper line under the AniSearch controls stating the merge rule in plain language: `Manuell > AniSearch > Jellyfin`.

### Required AniSearch states

- Idle state: AniSearch ID input, primary `AniSearch laden` button, helper text that the draft remains unsaved.
- Loading state: disable AniSearch input and load button, keep the button label in loading form, and leave manual field editing available.
- Success state: render a visible summary card immediately below the source-action row.
- Duplicate state: replace the normal success summary with a conflict card that names the existing anime and offers a direct edit-route action.
- Error state: show an inline error box inside the AniSearch card; do not clear the existing manual draft or Jellyfin-derived draft.

### Success summary contract

- The summary must be card-based, not only a toast or a page-level flash.
- The summary header must state that AniSearch updated the draft and that nothing is saved yet.
- Include three visible groups in the summary:
- `Aktualisierte Felder` listing the concrete fields AniSearch changed in this load.
- `Relationen` listing applied relation counts and any skipped or warning notes.
- `Entwurfsstatus` reminding the operator that save still happens only through `Anime erstellen`.
- If AniSearch overwrote Jellyfin-derived values, state that explicitly in the summary body.
- If AniSearch preserved existing manual values, state that explicitly in the summary body.

### Duplicate handling contract

- When the backend reports that the AniSearch ID already belongs to an existing anime, show a conflict card in the AniSearch area instead of a generic page error.
- Conflict copy must name the existing local anime title and AniSearch ID.
- Provide one direct secondary-or-primary redirect action labeled `Zum vorhandenen Anime wechseln`.
- Do not offer a parallel "force create anyway" path.

### Form behavior contract

- AniSearch provisional lookup text is not treated as protected manual content on create.
- Manual edits made after AniSearch loads remain authoritative and must not be visually marked as provider-owned.
- Later Jellyfin preview loads must not visually or semantically demote AniSearch-applied values; the summary should continue to communicate that AniSearch stays ahead of Jellyfin until the operator edits fields manually.
- AniSearch feedback belongs near the source controls, not buried in the bottom success message that appears after create persistence.

### Accessibility and responsiveness

- Keep all interactive controls keyboard reachable in the existing form order: title input, AniSearch ID input, AniSearch button, Jellyfin button.
- Use `aria-live="polite"` for AniSearch summary and status messaging, matching the existing edit-route AniSearch pattern.
- Maintain 44px minimum control height for AniSearch and Jellyfin action buttons on desktop and mobile.
- On screens under `720px`, stack AniSearch input, AniSearch button, and Jellyfin button vertically with full-width actions.
