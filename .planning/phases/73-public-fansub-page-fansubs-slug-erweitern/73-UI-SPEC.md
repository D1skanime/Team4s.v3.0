## Design System

| Property | Value |
|----------|-------|
| Tool | none (maßgeschneidertes internes CSS-Variablen-System) |
| Preset | nicht anwendbar |

verboten, wenn ein Pendant im Primitive-Set existiert. Altlasten in `FansubProfileTabs.tsx`
(native `<button>` Zeilen 118–131) MÜSSEN beim Umbau auf `Button`-Primitive migriert werden.

**shadcn-Gate:** `components.json` wurde nicht gefunden. Das Projekt verwendet kein shadcn — das
interne Primitive-Set (`@/components/ui`) erfüllt dieselbe Funktion. Kein shadcn-Init nötig.

## Color

Quelle: Verifiziert aus `ui.module.css`, `FansubProfileTabs.module.css`, `page.module.css` CSS-Variablen.

| Rolle | Value / Token | Verwendung |
|-------|--------------|------------|
| Dominant (60%) | `var(--color-white)` / `#f4f7fb–#e6ebf3` Gradient | Seitenhintergrund, Card-Flächen, Panel-Körper |
| Sekundär (30%) | `rgba(56, 70, 96, 0.07)` Border / `#e6ebf3` Card-BG | Karten-Ränder, Sticky-Nav-Hintergrund, Chip-Leiste |
| Akzent (10%) | `var(--color-primary)` / `var(--accent-primary)` | Reserviert für: aktive Sticky-Nav-Chips, `tabButtonActive`-Zustand, Timeline-Linke-Linie, externe Links, primäre Buttons |
| Gedämpft (historisch) | `var(--text-muted)` / `#66758d` | Historische/unbestätigte Nennungen, Empty-State-Text |
| Destructive | `var(--button-danger-*)` | Nicht in Phase 73 aktiv (read-only Seite, keine destruktiven Aktionen) |

**Akzent reserviert für (vollständige Liste):**
1. Aktiver Abschnitts-Chip in der Sticky-Nav (Desktop und Mobil)
2. Timeline-Vertikallinie im `GroupLeaderTimeline`-Abschnitt
3. Externe Links (Website, Deep-Dive-Verlinkungen)
4. Primary-`Button`-Primitive (z. B. falls CTA in zukünftigen Erweiterungen)

**Aktive Mitglieder-Karten:** Farbige Avatar-Karten (CONTEXT.md D-08) nutzen `Card`-Variante
`section` oder `elevated` — der Gradient enthält leicht Akzent-Blau (`cardSection`-Klasse in
`ui.module.css` Zeile 211: `color-mix(in srgb, var(--accent-primary) 10%, transparent)`).

**Historische Nennungen (D-09):** Grau, kleiner (14px), flach, ohne Avatar-Bild.
CSS-Klasse: gedämpfte Darstellung via `var(--text-muted)` und `Card`-Variante `flat` oder kein Card.
Badge `variant="muted"` aus `@/components/ui/Badge` für Label „unbestätigt".

**Gedenk-Block (D-08):** Würdevoller, ruhiger Sonderbereich. Keine Aktivitäts-/Mengen-Badges.
Keine Akzentfarbe — neutrales Grau (`badgeNeutral` oder schlicht ohne Badge).

Quelle: ui.module.css VERIFIED; CONTEXT.md D-08, D-09.

## Interaktionsverträge

### Sticky-Nav (D-04)

**Desktop:**
- `position: sticky; top: <globale Header-Höhe>` (CSS-Variable aus bestehendem Layout)
- Horizontale `flex`-Reihe mit Anker-Links zu: `#story`, `#highlights`, `#projekte`, `#team`,
  `#mitwirkende`, `#medien`, `#timeline`, `#deep-dive`
- Aktiver Abschnitt wird via `IntersectionObserver` erkannt und mit Akzentfarbe hervorgehoben
- Implementierung als `'use client'`-Komponente (`FansubSectionNav.tsx`)

**Mobil:**
- `overflow-x: auto; white-space: nowrap` — horizontal scrollbare Chip-Leiste
- Dieselben Anker, dieselbe Highlight-Logik
- Kein Scrollen zum aktiven Chip (kein JavaScript-`scrollIntoView` nötig für v1)
- Gap: 8px zwischen Chips

**Zustandsübergänge:**
- Inaktiver Chip: `Button` variant `ghost`
- Aktiver Chip: `Button` variant `subtle` (Akzentfarbe 10% Hintergrund) + `aria-current="true"`
- Transition: 120ms ease (erbt aus `.button`-Klasse)

### Mitglieder-Untergruppen (D-08/D-09)

- Aktive Mitglieder: Avatar-Kachel-Raster, 3 Spalten Desktop / 1 Spalte Mobil
- Ehemalige: kompaktere Liste, schwächere Schrift
- Historische Nennungen: Nur Text + `Badge variant="muted"` — kein Avatar, kein Card-Wrapper
- Gedenken: Eigener Unterbereich mit Trenner, keine Aktivitäts-Badges, kein Avatar-Bild

### Projektkarten (D-13)

- Link-Ziel: `/anime/[id]/group/[groupId]`
- Hover: `Card` variant `interactive` (translateY(-2px) + stärkerer Schatten — bereits in ui.module.css)

### Verlinkung historischer Mitglieder (D-10)

- `member_slug !== null` (geclaimt): Wrapper `<Link href="/members/{slug}">` — Name klickbar
- `member_slug === null` (ungeclaimt): Nur `<span>` — kein Link, kein Cursor-Pointer

### Empty States (D-15)

- Jeder Abschnitt bleibt in der DOM präsent (Anker stabil)
- `EmptyState` variant `compact` mit phasenspezifischem Text (siehe Copywriting)
- Keine Section wird bei leerem Datenstand aus der Sticky-Nav entfernt

## Responsive Verhalten (D-06)

| Breakpoint | Verhalten |
|-----------|-----------|
| < 640px (Mobil) | Alle Abschnitte einspaltig gestapelt; Sticky-Nav als horizontale scrollbare Chip-Leiste |
| 640px–1024px (Tablet) | Projektraster 2 Spalten; Mitglieder-Aktiv 2 Spalten; Medien-Bereiche 1 Spalte |
| > 1024px (Desktop) | Projektraster 3 Spalten; Mitglieder-Aktiv 3 Spalten; Medien 3 Bereiche nebeneinander; Sticky-Nav als horizontale Zeile |

Medien-Abschnitt (D-11): Drei Ownership-Bereiche auf Desktop nebeneinander (3-Spalten-Grid),
auf Mobil gestapelt (1 Spalte). Implementierung via CSS Grid `repeat(auto-fit, minmax(220px, 1fr))`.

Lesebreite (D-03): Fließtext-Abschnitte (Story, Highlights-Beschreibung) maximal 780px breit,
zentriert. Raster-Abschnitte (Projekte, Medien) nutzen die volle 1280px-Seitenbreite.
