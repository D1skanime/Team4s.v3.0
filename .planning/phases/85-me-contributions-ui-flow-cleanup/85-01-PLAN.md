<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI -> Team4s API | Protected contribution actions must continue through central `authorizedFetch` helpers, never raw token handling |
| Contribution domain -> Claim domain | Contribution components must not mutate or review Claim records |
| Anime-wide contribution -> release-version contribution | UI must not encode release-scoped work as anime-wide data |
| Modal overlay -> background page | Keyboard focus must stay inside modal while open |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-01 | Elevation of Privilege | Contribution components importing Claim APIs | mitigate | Remove Claim from `ReportModal` choices and add source grep verification for Claim API imports |
| T-85-02 | Tampering | Raw protected fetch in contribution UI | mitigate | Preserve `frontend/src/lib/api.ts` helpers only; grep for raw `fetch`/`authorizedFetch` in components |
| T-85-03 | Tampering | Release-version work submitted as anime-wide contribution | mitigate | Keep release-version option disabled/unavailable until real selector/contract exists; test payload omits `release_version_id` |
| T-85-04 | Spoofing/Accessibility | Focus escapes modal and user activates background action | mitigate | Global Modal focus trap, Escape handling, focus return tests |
| T-85-05 | Information Disclosure | Generic report center exposes unrelated Claim context in contributions | mitigate | Claim no longer peer option; blocked identity state links to profile/account setup only |
| T-85-SC | Supply Chain | New dependency for modal/year controls | accept | No new package. Reuse existing project helpers/components |
</threat_model>
