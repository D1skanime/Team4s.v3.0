## Phase Overview

Phase 6 decomposes the current `episode_versions` model into clearer release-layer concepts without breaking the stable public API during the first rollout.

Today, `episode_versions` carries several responsibilities at once:
- release identity
- fansub/group association
- variant properties like subtitle/video quality
- provider/source stream data
- read ordering for public release views

This phase separates those concerns into explicit schema layers while preserving current handlers and contracts through repository adapters.

## Contract Status

**Contract default: FREEZE for first rollout**

No public API changes should be required for the initial decomposition slice if adapters preserve:
- grouped episode version reads
- release detail reads
- release stream grant flows
- group releases listing

If new semantics require new public shapes later, that becomes a named future slice after repository parity is proven.

---

## Target Model (Phase 6 slice)

Introduce explicit tables for:
1. **release identities**
2. **release versions**
3. **release variants**
4. **release streams / provider sources**

Suggested decomposition:
- `releases`
  - stable release identity per anime/group/episode context
- `release_versions`
  - release-level version metadata and naming/timestamps
- `release_variants`
  - subtitle/video/audio distinctions
- `release_streams`
  - provider/source/external stream resolution data

Exact names may be adjusted during schema package work, but the split of responsibilities should remain.
