<objective>
Release-Theme-Asset-Uploads persistieren die tatsaechliche Dateigroesse in der media_files-Tabelle, sodass die List-API size_bytes mit dem echten Wert zurueckgibt statt immer 0.

Purpose: Der Bug liegt darin, dass beide Upload-Handler nach CreateMediaAsset keinen media_files-INSERT ausfuehren, obwohl die ListReleaseThemeAssets-Query size_bytes per COALESCE-Subquery aus media_files liest. Ohne Eintrag in media_files liefert COALESCE 0.

Output:
- Neue Methode InsertMediaFile auf *repository.MediaRepository (per D-04)
- Beide Upload-Handler erweitert um InsertMediaFile-Aufruf nach CreateMediaAsset (per D-05)
- Rollback bei InsertMediaFile-Fehler: DeleteMediaAsset + removeFileQuietly (per Discretion)
- Source-Text-Tests die FIX-01, FIX-02, FIX-03 absichern
</objective>

    ```go
    // InsertMediaFile legt einen media_files-Eintrag fuer ein per CreateMediaAsset
    // erstelltes Asset an. width und height werden als 0 gesetzt (fuer Video-Assets).
    // Wird verwendet wenn kein MediaUploadRepository verfuegbar ist.
    func (r *MediaRepository) InsertMediaFile(
        ctx context.Context,
        mediaID int64,
        variant string,
        path string,
        size int64,
    ) error {
        _, err := r.db.Exec(ctx, `
            INSERT INTO media_files (media_id, variant, path, width, height, size)
            VALUES ($1, $2, $3, 0, 0, $4)
        `, mediaID, variant, path, size)
        if err != nil {
            return fmt.Errorf("insert media file for asset %d: %w", mediaID, err)
        }
        return nil
    }
    ```
