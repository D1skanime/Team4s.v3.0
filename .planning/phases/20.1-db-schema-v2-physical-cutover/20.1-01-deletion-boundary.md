# Phase 20.1 Plan 01 Deletion Boundary

## Allowed Future Drop Targets

Only these legacy release-bearing episode-version tables are allowed drop targets in later Phase 20.1 migration plans:

- `episode_version_episodes`
- `episode_version_images`
- `episode_versions`

They are legacy persistence for the old flat episode-version model. The local reset script clears their disposable test rows but does not drop them.

## Tables That Must Remain

These target identity, release graph, lookup, and media tables must not be dropped as part of the legacy episode-version cleanup:

- `episodes`
- `episode_titles`
- `episode_types`
- `languages`
- `episode_filler_types` once created
- `fansub_releases`
- `release_sources`
- `release_versions`
- `release_variants`
- `release_variant_episodes` once created
- `release_version_groups`
- `release_streams`
- `stream_sources`
- `stream_types`
- `visibilities`
- `media_assets`
- `media_external`
- `media_files`
- `anime_media`
- `episode_media`
- `release_media`

## Operational Rule

Later destructive cleanup must target tables by exact name. Broad patterns such as `episode_*`, release graph table families, or media table families are outside the approved deletion scope.
