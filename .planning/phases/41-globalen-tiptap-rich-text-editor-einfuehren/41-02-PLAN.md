    **ValidateJSON:** Unmarshal + validateNode (rekursiv). Prüft Node-Typ, Mark-Typen, colorToken-Wert,

    Fehlertexte:
    - "nicht erlaubter Node-Typ: %q"
    - "nicht erlaubter Mark-Typ: %q"
    - "nicht erlaubtes Farb-Token: %q"
    - "Tabelle überschreitet maximum von 30 Zeilen"
    - "Tabellenzeile überschreitet maximum von 6 Spalten"
    - "Verschachtelte Tabellen sind nicht erlaubt"

    **Tabellen-Struktur-Validierung im rekursiven Walker** (in validateNode implementieren):

    ```go
    case "table":
        // Zeilen-Limit prüfen
        rowCount := 0
        for _, child := range node.Content {
            if child.Type == "tableRow" {
                rowCount++
            }
        }
        if rowCount > 30 {
            return fmt.Errorf("Tabelle überschreitet maximum von 30 Zeilen")
        }

    case "tableRow":
        // Spalten-Limit prüfen
        colCount := 0
        for _, child := range node.Content {
            if child.Type == "tableCell" || child.Type == "tableHeader" {
                colCount++
            }
        }
        if colCount > 6 {
            return fmt.Errorf("Tabellenzeile überschreitet maximum von 6 Spalten")
        }

    case "tableCell", "tableHeader":
        // Keine verschachtelten Tabellen
        for _, child := range node.Content {
            if child.Type == "table" {
                return fmt.Errorf("Verschachtelte Tabellen sind nicht erlaubt")
            }
        }
    ```

    Diese Prüfungen laufen VOR dem rekursiven Abstieg in node.Content, damit der Fehler
    so früh wie möglich zurückgegeben wird. Nach den Struktur-Prüfungen wird node.Content
    normal rekursiv validiert (Allowlist-Check für Kind-Nodes).

    **RenderHTML:** Unmarshal + renderNode (rekursiv) → strings.Builder → sanitizer.SanitizeBytes.
    Node-Mapping:
    - doc → kein eigener Tag, nur Content rendern
    - paragraph → `<p>...</p>`
    - text → text mit Marks gewrappt; EscapeString aus html/template für den Text-Inhalt
    - heading level 1-3 → `<h1>`/`<h2>`/`<h3>` (level aus Attrs["level"].(float64), default 1, clamp 1-3)
    - bulletList → `<ul>...</ul>`
    - orderedList → `<ol>...</ol>`
    - listItem → `<li>...</li>`
    - blockquote → `<blockquote>...</blockquote>`
    - horizontalRule → `<hr>`
    - table → `<table>...</table>`
    - tableRow → `<tr>...</tr>`
    - tableHeader → `<th>...</th>` (mit colspan/rowspan aus Attrs wenn vorhanden)
    - tableCell → `<td>...</td>` (mit colspan/rowspan aus Attrs wenn vorhanden)

    Marks auf text-Nodes:
    - bold → `<strong>...</strong>`
    - italic → `<em>...</em>`
    - textStyle mit colorToken → `<span data-color-token="TOKEN" class="color-token-TOKEN">...</span>`
      Nur wenn colorToken vorhanden und nicht "default".
    Marks werden von innen nach außen gewrappt (innerster Wrapper zuerst).

    **ExtractText:** Rekursiv alle text-Node-Texte sammeln, durch Leerzeichen getrennt. strings.TrimSpace am Ende.

    **IsEmpty:** ExtractText aufrufen, strings.TrimSpace("") prüfen. Bei Fehler → true.

    **bluemonday Policy:**
    ```go
    func newTipTapSanitizerPolicy() *bluemonday.Policy {
        p := bluemonday.NewPolicy()
        p.AllowElements("p", "h1", "h2", "h3", "strong", "em",
            "ul", "ol", "li", "blockquote",
            "table", "thead", "tbody", "tr", "th", "td", "hr", "span")
        p.AllowAttrs("class").OnElements("span", "td", "th")
        p.AllowAttrs("colspan", "rowspan").OnElements("td", "th")
        p.AllowAttrs("data-color-token").OnElements("span")
        return p
    }
    ```
