<threat_model>
Assets: Team4s protected API endpoints, Keycloak JWT trust boundary, app-user resolver, revocation state.
Threats:
- HIGH: ID tokens remain accepted because verifier still checks frontend client audience.
- HIGH: Access tokens with wrong audience or wrong authorized party are accepted.
- HIGH: Signature/issuer/expiry validation is accidentally weakened when moving away from `oidc.IDTokenVerifier`.
- MEDIUM: Logout token verification breaks if it shares the API-audience verifier.
Mitigations: add explicit access-token test fixtures, preserve JWKS signature checks, validate `iss`/`exp`/`aud`/`azp`, and keep logout-token verification as a separate method/path.
</threat_model>
