<threat_model>
Assets: Keycloak-issued browser tokens, Team4s API access boundary, refresh sessions.
Threats:
- HIGH: Access tokens omit `team4s-api`, causing implementers to keep accepting ID tokens.
- HIGH: Audience mapper is overbroad or attached to ID tokens, weakening token separation.
- MEDIUM: Existing local Keycloak volumes hide realm import changes and make smoke tests misleading.
Mitigations: assert `aud` on decoded access tokens, assert ID token remains `aud=team4s-frontend`, document fresh-import and live-update/reset paths, and keep domain authorization out of Keycloak claims.
</threat_model>
