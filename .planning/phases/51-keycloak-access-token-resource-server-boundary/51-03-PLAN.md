<threat_model>
Assets: browser-stored bearer token, refresh token, Team4s API calls, current-user identity.
Threats:
- HIGH: Frontend continues persisting `idToken` under `team4s_access_token`.
- HIGH: Refresh path returns ID token, causing later requests to regress after the first refresh.
- MEDIUM: Tests assert generic token strings and miss the ID/access distinction.
- MEDIUM: Token changes leak raw token ownership into pages/components.
Mitigations: test with distinct `fresh-access-token` and `fresh-id-token`, assert cookies/headers use access token only, and keep changes inside central auth files.
</threat_model>
