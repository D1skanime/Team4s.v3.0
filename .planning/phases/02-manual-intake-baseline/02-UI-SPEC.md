| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon/text offsets, tight chip spacing |
| sm | 8px | Field label-to-input gaps, chip gaps, inline actions |
| md | 16px | Default field spacing, save-bar internals, panel stacks |
| lg | 24px | Card padding, cover section gaps, section spacing |
| xl | 32px | Page section gaps, top-level header spacing |
| 2xl | 48px | Page bottom padding, large section breaks |
| 3xl | 64px | Reserved for wide desktop whitespace only; do not use inside the form body |

Exceptions: none. Touch targets must still meet a 44px minimum interactive height, but that accessibility requirement is not a spacing token. Sticky save bar offset uses the declared spacing scale only: 16px on mobile and 16px on desktop.

## Screen Contract

### Entry

- The intake entry point for this phase must expose exactly one manual primary action labeled `Neu manuell`.
- The manual option must explain the minimum contract in one sentence: title and cover are enough to create the first record.
- Jellyfin entry UI, provenance controls, and relation UI are out of scope and must not appear on the manual-create screen.

### Layout

- Preserve one shared editor surface across create and edit by reusing the Phase 1 shell, central save bar, and admin CSS-module styling language.
- Use a content-first stack: page header, one main form panel, and the sticky save bar.
- Make the required `title + cover` block the primary visual anchor of the screen; the sticky save bar is the secondary operational emphasis.
- On desktop, keep form fields in responsive two-column groups only where scan speed improves; title, cover, validation summary, and success messaging stay full-width.
- On mobile, collapse to one column with no horizontal overflow and maintain 44px minimum control height.

### Draft And Preview-Before-Save

- Manual create remains a client-side draft until the admin explicitly saves; no anime ID, ownership badge detail, or studio-only metadata should appear before save.
- Show an unsaved draft preview inside the form, not in a separate route. The preview includes current title text and current cover preview, with placeholders when either is missing.
- The save bar must use three explicit states:
  1. `Noch kein manueller Entwurf` when no meaningful input exists.
  2. `Entwurf unvollstaendig` when there are changes but either title or cover is missing.
  3. `Entwurf bereit zum Anlegen` when title and cover are both present.
- The save button stays visually primary at all times but remains disabled until the draft reaches state 3.

### Form Structure

- Put required fields first: title, cover.
- Place optional editorial fields after the minimum contract: type, content type, status, year, max episodes, localized titles, genres, description.
- Use visible labels for every input. Do not rely on placeholders as labels.
- Keep advanced-but-still-manual metadata in progressive disclosure order: core identity, cover, optional taxonomy, optional long-form description.

### Validation And Feedback

- Validate required fields inline on blur and again on submit.
- Show field-local errors directly below the relevant control and a single summary message above the form when submit fails.
- Error copy must state the problem and the next action. Avoid raw backend field names in the default UI copy.
- When cover upload is running, disable the upload trigger and show clear loading text in-place. Do not freeze unrelated text inputs.
- When final save is running, disable the primary save action, keep the draft visible, and change the button label to `Anime wird erstellt...`.

### Success Transition

- After successful create, show a success confirmation in the save bar or top banner using the created anime number if available.
- Redirect immediately after success to the studio/editor context for the new anime so the admin lands in the same shared editor surface for continued maintenance.
- The transition should feel operational, not celebratory: no decorative animation, confetti, or modal interruption.

### Touch, Focus, And Motion

- All interactive controls must meet 44px minimum height or hit area.
- Use the existing accent color for focus visibility on inputs, buttons, and the active cover-upload target.
- Motion is limited to meaningful state transitions under 150ms for hover/focus/press; no decorative animation is permitted in this phase.
