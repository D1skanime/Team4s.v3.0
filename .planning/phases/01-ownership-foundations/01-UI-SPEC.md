# Phase 1 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | repo-local CSS modules |
| Preset | not applicable |
| Component library | custom admin components and route-local primitives |
| Icon library | `lucide-react` |
| Font | `Inter, system-ui, sans-serif` |

Phase 1 will extend the existing admin anime workspace rather than introduce shadcn or Tailwind. Source baseline: `01-CONTEXT.md`, `01-RESEARCH.md`, `frontend/src/app/admin/anime/AdminStudio.module.css`, `frontend/src/app/admin/anime/components/AnimeEditPage/AnimeEditWorkspace.module.css`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight text offsets, helper/meta spacing |
| sm | 8px | Label-to-input gaps, chip gaps, badge gaps |
| md | 16px | Default field spacing, card internals, action gaps |
| lg | 24px | Section padding, card gaps, workspace rhythm |
| xl | 32px | Page spacing, large section separation |
| 2xl | 48px | Page bottom padding, large block separation |
| 3xl | 64px | Reserved for future full-page breaks only |

Exceptions: minimum interactive height is `44px` for buttons, inputs, selects, and primary click targets; compact badges may use `32px` minimum height; removable chips may use `36px` minimum height.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.3 |
| Display | 28px | 600 | 1.2 |

Use only weights `400` and `600` for new Phase 1 UI work. Existing `500` and `700` usages in the admin surface should be normalized toward this contract when touched by the phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f9f9f9` | Page canvas, muted panels, empty states, non-primary drop zones |
| Secondary (30%) | `#ffffff` | Cards, inputs, sticky save bar, dropdowns, image frames |
| Accent (10%) | `#ff6a3d` | Primary save CTA, focus borders, active upload/drop states, ownership-linked badges |
| Destructive | `#dc3545` | Destructive actions only |

Accent reserved for: global save button, keyboard focus treatment, drag-and-drop active state, selected ownership/source badges that indicate linked external context, and the top-priority action in a section. Do not use accent for every secondary button, helper link, or static metadata label.

## Interaction Contract

| Area | Contract |
|------|----------|
| Page shell | Reuse the current sectioned workspace and central save model. Create and edit must read as the same product surface with different entry context, not different layouts. |
| Ownership visibility | Add one record-level ownership badge row directly below the page subtitle. Show exactly one primary state badge: `Manuell gepflegt` or `Mit externer Quelle verknuepft`. Support one muted secondary hint for attribution context such as `Aenderungen werden deinem Admin-Konto zugeordnet`. |
| Section hints | Where ownership context matters in Phase 1, place a small muted or accent-tinted badge beside the section title, not per-field provenance icons. Full per-field and per-asset provenance remains out of scope until Phase 4. |
| Save behavior | Keep one sticky bottom save bar with left-side status copy and one right-side primary action. Save stays disabled when there are no dirty changes or while a request is in flight. Do not introduce per-section save buttons. |
| Validation | Show field-level errors directly below the related input and an error notice at the top of the workspace when save fails. On submit failure, focus the first invalid field. |
| Loading and async feedback | Preserve layout during async work. Buttons switch label state instead of resizing. Autocomplete and upload states stay inline inside the owning section. |
| Destructive actions | Separate destructive controls from the primary save action. Require explicit confirmation before irreversible editor-borne removals such as cover deletion. |
| Responsive behavior | Use 3 columns for dense metadata on desktop, collapse to 1 column below `992px`, and keep the sticky save bar stacked and readable below `768px`. Cover preview moves above actions on narrow widths. |
| Motion | Restrict motion to `120-200ms` border, background, shadow, and opacity transitions. No decorative animation. Respect `prefers-reduced-motion`. |

Primary focal point: the header card with the anime identity and ownership summary should draw the eye first; the sticky save CTA is the secondary focal point that becomes dominant only after the form is dirty.

## Component Inventory

| Component | Visual Contract | Interaction Contract |
|-----------|-----------------|----------------------|
| Header card | White card on muted page canvas, 24px padding, 28px page title, 16px subtitle | Shows page identity plus ownership summary row before the first editable section |
| Ownership badge row | 8px gaps, pill shape, muted/manual state and accent-linked state | Read-only in Phase 1; never hides save or validation controls |
| Section card | White card, 24px padding, 20px heading, 13px section meta | Groups related fields and keeps one concern per card |
| Form fields | 44px minimum height, 8px label gap, 8px radius, neutral border until focus | Inline validation, clear focus ring via accent border, no placeholder-only labels |
| Genre chips and autocomplete | Neutral pills, white dropdown, soft shadow, muted metadata text | Keyboard navigation, retry action inline, no modal interruption |
| Cover drop zone | Dashed neutral border by default, accent border plus soft tint on active drag | Click and drop both supported; upload state stays in the section |
| Sticky save bar | White translucent surface, 12px radius, light border and shadow | Persists at viewport bottom, stacks on mobile, single primary save action only |
| Notices | Neutral, error, and success boxes reuse the existing admin card language | Error notice explains the problem and next step; success notice remains brief |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Aenderungen speichern` |
| Empty state heading | `Manuell gepflegt` |
| Empty state body | `Fuer diesen Anime ist aktuell keine externe Quelle verknuepft. Bearbeite die Felder direkt und speichere den allgemeinen Anime-Kontext.` |
| Error state | `Aenderungen konnten nicht gespeichert werden. Pruefe die markierten Felder, deine Anmeldung und die Verbindung zum Server, und versuche es dann erneut.` |
| Destructive confirmation | `Cover entfernen`: `Dieses Cover wird sofort vom Anime geloest. Moechtest du das Cover wirklich entfernen?` |

Additional microcopy rules:
- Ownership badge copy must stay short and noun-based.
- Save-state copy must use a binary model only: saved vs unsaved.
- Avoid technical provenance vocabulary such as `normalized metadata`, `patch semantics`, or `sync overlay` in the Phase 1 UI.
