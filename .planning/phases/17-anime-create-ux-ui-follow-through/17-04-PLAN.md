# Plan 17-04: Unified Asset Section — Alle Assets in einem dedizierten Bereich

## Objective

Section 2 (`Assets`) bekommt einen einheitlichen Asset-Bereich. Alle Asset-Slots (Cover, Banner, Logo, Hintergründe, Background-Video) erscheinen als visuelle Karten mit Source-Badge, Vorschau und Aktionen. Jellyfin-Assets und manuell hochgeladene Assets koexistieren in denselben Slot-Karten. `JellyfinDraftAssets` verschwindet als separate Einheit und wird durch die neuen einheitlichen Karten ersetzt. Die bestehenden Upload- und Linking-Seams (`ManualCreateAssetUploadPanel`, `handleCoverUpload`, etc.) bleiben unverändert — sie werden nur in der neuen Struktur neu verdrahtet.

<action>
Neue Datei `frontend/src/app/admin/anime/create/CreateAssetSection.tsx` anlegen.

Die Komponente erhält alle staged Assets und Jellyfin Draft Assets als Props und rendert sie in einem einheitlichen Grid. Für jeden Slot wird `CreateAssetCard` verwendet.

Wichtige Props:
- `stagedCoverPreviewUrl?: string | null` — staged Cover Vorschau
- `stagedBanner?: StagedAsset | null`
- `stagedLogo?: StagedAsset | null`
- `stagedBackgrounds?: StagedAsset[]`
- `stagedBackgroundVideo?: StagedAsset | null`
- `jellyfinDraftAssets?: JellyfinDraftAssetSlots | null` — Jellyfin-Assets nach Übernehmen
- `onOpenFileDialog: (kind: AssetKind) => void`
- `onOpenAssetSearch: (kind: AssetKind) => void`
- `onRemoveSingleAsset: (kind: AssetKind) => void`
- `onRemoveBackground: (index: number) => void`
- `onRemoveJellyfinAsset: (kind: string) => void`
- `fileInputRefs: React.RefObject<...>` — für manuelle Upload-Inputs

Für jeden Slot (`cover`, `banner`, `logo`, `background`, `background_video`):
1. Prüfen ob staged Asset vorhanden → `source="Manuell"`
2. Falls kein staged Asset, prüfen ob Jellyfin-Draft-Asset vorhanden → `source="Jellyfin"`
3. Falls keins → leer anzeigen

Statusnote immer `"Wird beim Erstellen übernommen"` wenn ein Asset vorhanden ist.

Cover-Slot erhält `isRequired={true}`.

Multi-Background: als Tile-Grid mit + Hinzufügen-Karte.

Beispiel-Struktur (Typen aus existierendem Code übernehmen):
```tsx
export function CreateAssetSection({ ... }) {
  return (
    <div className={createStyles.assetGrid}>
      {/* Cover */}
      <CreateAssetCard
        label="Cover"
        isRequired
        previewUrl={stagedCoverPreviewUrl ?? jellyfinDraftAssets?.cover?.url}
        source={stagedCoverPreviewUrl ? "Manuell" : jellyfinDraftAssets?.cover ? "Jellyfin" : null}
        statusNote={stagedCoverPreviewUrl || jellyfinDraftAssets?.cover ? "Wird beim Erstellen übernommen" : undefined}
        isEmpty={!stagedCoverPreviewUrl && !jellyfinDraftAssets?.cover}
        actions={
          <>
            <button type="button" onClick={() => onOpenFileDialog("cover")}>Hochladen</button>
            <button type="button" onClick={() => onOpenAssetSearch("cover")}>Online suchen</button>
            {(stagedCoverPreviewUrl) ? (
              <button type="button" onClick={() => onRemoveSingleAsset("cover")}>Entfernen</button>
            ) : null}
          </>
        }
      />
      {/* Banner, Logo analog */}
      {/* Backgrounds: map über Array */}
      {/* Background Video */}
    </div>
  );
}
```

Die genauen Typen für `JellyfinDraftAssetSlots`, `StagedAsset`, `AssetKind` aus den bestehenden Dateien übernehmen — NICHT neu erfinden.
</action>
