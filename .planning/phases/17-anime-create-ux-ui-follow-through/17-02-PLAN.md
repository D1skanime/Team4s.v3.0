# Plan 17-02: AniSearch Card — Framing als Metadaten-Quelle + Copy-Bereinigung

## Objective

Die AniSearch-Card wird als explizite Metadaten-Quelle gerahmt: Heading, Intro-Text und alle Status-Meldungen verwenden die genehmigte temporale Copy aus dem UI-Spec. Alle „Entwurf"-Formulierungen, die das falsche Draft-Produkt-Modell implizieren, werden durch präzise temporale Alternativen ersetzt. Die existierende Funktionalität (ID-Eingabe, Titelsuche, Kandidatenliste, Laden) bleibt vollständig erhalten.

---

### Task 2: createAniSearchSummary.ts — Status-Meldungen ersetzen

<read_first>
- frontend/src/app/admin/anime/create/createAniSearchSummary.ts (vollständig)
</read_first>

<action>
In `createAniSearchSummary.ts`:

Zeile 81 — die Meldung:
```
`AniSearch ID ${result.anisearch_id} hat den Entwurf aktualisiert. Noch nichts gespeichert.`
```
ersetzen durch:
```
`AniSearch ID ${result.anisearch_id} geladen. Wird beim Erstellen übernommen.`
```

- `"... noch nicht gespeichert"` → `"... Wird beim Erstellen übernommen."`
</action>

<acceptance_criteria>
