# Phase 71 — UI Design Contract

> Visual and interaction contract for: UI-Politur Fansub-Contributions und Member-Profil auf globales Design-System.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a brownfield phase: the contract is bound to the EXISTING custom design system, not a new one. Executors MUST consume existing tokens from `frontend/src/styles/globals.css` and primitives from `@/components/ui`. No new tokens, no new primitive types, no new colors.

**Mandatory primitive constraint (CLAUDE.md "Frontend-UI" + ESLint `uiPrimitiveGuard`):**
Every Phase-71 surface MUST use `@/components/ui` primitives. Native `<select>`/`<input>`/`<textarea>`/`<button>` are forbidden for any primitive type the barrel already offers. Local file consistency does NOT justify deviating. The two native `<select>` in `AnimeContributionModal.tsx` (lines 349/365) MUST become `Select` + `FormField`. ESLint guard stays at `warn` for this phase (D-14) — migration scope is Phase-71 surfaces only (D-13).

**German string quality (CLAUDE.md "Sprachqualität"):**
All user-facing DE strings in this phase MUST use correct umlauts (ä ö ü Ä Ö Ü ß). ASCII substitutions (ae/oe/ue/ss) are forbidden in JSX text, labels, aria-labels, placeholders, error/toast copy. (Note: this UI-SPEC uses ASCII in prose for tooling safety, but shipped strings must use real umlauts.)

## Spacing Scale

Existing `globals.css` `--space-*` tokens. Executors MUST reference these variables, never raw px. All values are multiples of 4.

| Token | Variable | Value | Usage |
|-------|----------|-------|-------|
| 1 | `--space-1` | 4px | Icon-to-label gap inside a Badge, inline chip padding |
| 2 | `--space-2` | 8px | Compact element spacing, chip row gaps, badge gaps |
| 3 | `--space-3` | 12px | Form field inner spacing |
| 4 | `--space-4` | 16px | Default element spacing, FormField vertical rhythm |
| 5 | `--space-5` | 24px | Card / section padding |
| 6 | `--space-6` | 32px | Layout gaps, page-shell vertical padding |
| 7 | `--space-7` | 48px | Major section breaks |
| 8 | `--space-8` | 64px | Page-level spacing |

Control heights (touch targets) come from tokens: `--control-height-sm` 36px, `--control-height-md` 44px, `--control-height-lg` 52px. The migrated `Select` in the contribution modal inherits the primitive's height — do not hardcode.

Exceptions: none. The migrated media image (D-16) uses an aspect-driven box, not a spacing token; ratio is fixed (recommend 16:9) with `object-fit: cover` and BOTH width and height set to kill the Next/Image "width or height modified, but not the other" warning.

## Typography

Single body scale from `globals.css` (`body { font-size: 16px; line-height: 1.5 }`). No new heading sizes are introduced by this phase; section/eyebrow styles already exist. Declared roles in use:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label / eyebrow | 14px | 600 | 1.2 (uppercase, letter-spacing 0.08em via `.eyebrow`) |
| Section heading | 20px | 600 | 1.2 (via `SectionHeader` primitive) |
| Card / page heading | 28px | 600 | 1.2 (via `PageHeader` primitive) |

Weights: regular (400) for body, semibold (600) for labels/headings. Exactly two weights. Executors take headings via `PageHeader`/`SectionHeader` primitives rather than ad-hoc font sizes.

## Color

Existing semantic tokens from `globals.css`. No new color values are introduced. The 60/30/10 split maps onto existing surface/text/accent tokens.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--bg-primary` / `--color-bg-light` | #f9f9f9 | Page background, app canvas |
| Secondary (30%) | `--bg-card` / `--surface-card` | #ffffff | Cards, modal surface, tabs, tables, profile panels |
| Accent (10%) | `--color-primary` | #5f84dd | Primary actions only (see reserved list) |
| Destructive | `--color-error` | #dc3545 | Destructive actions / error text only |

Accent (#5f84dd) reserved for: the primary CTA inside `AnimeContributionModal` (Speichern/Mitwirkenden hinzufuegen), the per-anime-row "Mitwirkende" action trigger in the `releases` tab, active `Tabs` indicator, and focused form controls. NOT for badge fills, NOT for chips, NOT for every link.

Success-badge color coding (D-12) uses the EXISTING `Badge` variant palette — do not invent hex values. Map each badge_code to one of the six existing `Badge` variants:

| badge_code | Badge variant | lucide-react icon (Claude discretion, recommended) |
|------------|---------------|----------------------------------------------------|
| `founding_member` | `warning` (gold) | `Crown` |
| `historical_leader` | `info` (blue) | `Shield` |
| `long_term_member` | `success` (green) | `CalendarClock` |
| `first_contribution` | `neutral` | `Sparkles` |
| `verified` | `success` (green) | `BadgeCheck` |
| `productive_bronze` / `silver` / `gold` (if rendered) | `muted` / `neutral` / `warning` | `Layers` |
| `all_rounder` (if rendered) | `info` | `Hexagon` |

Icon sits before the label inside the `Badge`, gap `--space-1`. Replace the current text-glyph prefixes (★ ♦ ◆ ✦ ✓) in `BADGE_LABELS` with `lucide-react` icons.

## Copywriting Contract

All German. Real umlauts mandatory in shipped code.

| Element | Copy |
|---------|------|
| Section label (renamed) | "Mitwirkende" — replaces "Beitraege"/"Anime-Beitraege" EVERYWHERE (admin edit, public anime page, public group/member profiles, all counters/labels) per D-06. Counter form: "{n} Mitwirkende". |
| Per-row action trigger (releases tab) | "Mitwirkende" (opens existing `AnimeContributionModal` per D-05) |
| Primary CTA (contribution modal) | Keep the modal's existing save verb-noun; render via `Button` primary variant. No new label invented. |
| Empty state — public anime contributors | NONE. Section is fully hidden when an anime has no contributors (D-09). Do NOT render a placeholder or `EmptyState`. |
| Empty state — role timeline (`MemberRoleTimeline`) | NONE. Section is fully hidden when there are no roles (D-17). Remove the loose "-" filler. |
| Error state — badge visibility mutation | Existing copy retained: "Badge konnte nicht ausgeblendet werden. Bitte versuche es erneut." (lives in `/me/profile` after D-10 move, not on the public page) |
| Error state — generic load failure | Use `ErrorState` primitive with problem + retry path; reuse existing project error copy patterns, no new invented strings. |
| Destructive confirmation | None in scope. Badge "Ausblenden" is a reversible visibility toggle (not destructive) and moves to `/me/profile` edit context (D-10); no confirmation dialog required. No delete/destructive action is introduced by this phase. |

Contributor sort order (public display, D-08): group by role, lead/most-important roles first (e.g. `project_lead`, `editor`), alphabetical by name within the same role.
