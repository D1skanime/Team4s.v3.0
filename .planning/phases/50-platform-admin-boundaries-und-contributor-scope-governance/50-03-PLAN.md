<objective>
Implement the frontend route separation: guard global admin pages for platform admins, rebuild `Meine Gruppen` as the non-admin group-management entry, and retire or redirect the current `/admin/my-groups` placeholder implementation.

Purpose: the product should look like the permission model actually works.
</objective>

<execution_strategy>
This plan should be executed as parallel implementation slices when agent
capacity is available. Do not let one agent own the whole UI refactor.

Preferred slices:
- route guards and navigation
- `Meine Gruppen` / scoped group route
- `Grunddaten` platform-admin versus lead-safe split
- `Medien` UI and persistence wiring
- `Mitglieder` app-user assignment/invitation UX
- `Gruppengeschichte` editor shell
- profile `Mitgliedergeschichte` editor shell
- release-version media/notes contributor tab gating

Each slice must leave notes about touched auth/session state, API calls,
capability dependencies, and saved-data paths so Phase 50-04 reviewers can
verify the combined system.
</execution_strategy>
