<review_strategy>
Run separate review passes, preferably by separate review agents when available:
- Auth reviewer: route guards, backend authorization, direct URL/API denial,
  Keycloak identity-only boundary, 401/403 behavior.
- DB/persistence reviewer: no new accidental tables, no migration-chain risk,
  writes land in the intended existing tables, saved values reload correctly.
- Frontend state/API reviewer: central auth/API client usage, capability refresh,
  stale state after mutations, session/user switch behavior.
- UI reviewer: global Anime-Einblicke/editor pattern reuse, scoped group sections,
  release-version `Medien`/`Notizen` contributor UI, profile story UI, minimal
  helper text, responsive layout.
- Domain reviewer: fansub group, profile, anime, release, and media ownership stay
  on the correct entities.

The phase should not close until each review pass records findings or explicitly
states no blocking issues for its risk area.
</review_strategy>
