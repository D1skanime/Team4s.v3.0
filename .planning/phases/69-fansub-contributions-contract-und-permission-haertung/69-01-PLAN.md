  <files>database/migrations/0088_anime_contributions_constraints.up.sql</files>
  <action>
  Erstelle database/migrations/0088_anime_contributions_constraints.up.sql.

  Dateikommentar: "Migration 0088: Cross-Group-Schutz und Duplikat-Schutz fuer anime_contributions."

  Schritt 1 — Hilfs-Unique auf hist_fansub_group_members:
  Damit der Composite-FK auf (fansub_group_id, id) moeglich ist, braucht PostgreSQL einen
  eindeutigen Index auf exakt diesen Spalten. Die bestehende UNIQUE(fansub_group_id, member_id)
  deckt das NICHT ab. Fuege einen neuen Unique-Constraint hinzu:

      ALTER TABLE hist_fansub_group_members
          ADD CONSTRAINT uq_hist_fansub_group_member_group_id
          UNIQUE (fansub_group_id, id);

  Schritt 2 — Composite-FK auf anime_contributions:
  Jetzt kann der FK referenziert werden. Fuer den FK braucht anime_contributions beide
  Spalten explizit im FK-Statement:

      ALTER TABLE anime_contributions
          ADD CONSTRAINT fk_anime_contributions_member_group
          FOREIGN KEY (fansub_group_id, fansub_group_member_id)
          REFERENCES hist_fansub_group_members(fansub_group_id, id)
          ON DELETE RESTRICT;

  Schritt 3 — Unique-Constraint gegen Duplikate:
  Verhindert, dass derselbe Member zweimal fuer dasselbe (Gruppe, Anime)-Paar eingetragen wird:

      ALTER TABLE anime_contributions
          ADD CONSTRAINT uq_anime_contribution_member
          UNIQUE (fansub_group_id, anime_id, fansub_group_member_id);

  Reihenfolge: Zuerst Schritt 1, dann Schritt 2, dann Schritt 3.
  Jede ALTER TABLE-Anweisung als eigene SQL-Zeile, abgeschlossen mit Semikolon.
  </action>
