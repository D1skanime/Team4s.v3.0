# Phase 83 — UI Design Contract

> Visual and interaction contract for the Pro-Release-Mitwirkenden-Zuordnung (release_version_id) im Cockpit.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Source of truth for tokens: `frontend/src/styles/globals.css` (`:root`) + `frontend/src/components/ui/ui.module.css`.
> All values pre-populated from the existing codebase token system and locked CONTEXT decisions (D-01…D-16). No new design questions were required.

## Hard Constraints (non-negotiable, gate the whole phase)

| ID | Constraint | Source |
|----|-----------|--------|
| **C-UI-PRIMITIVES** | Every user-facing control MUST use `@/components/ui` primitives (`Button`, `Select`, `FormField`, `Modal`, `Drawer`, `Card`, `Table`, `Badge`, `Input`, `Textarea`, `Tabs`) plus existing shared components. Hand-built native `<select>/<input>/<textarea>/<button>` or bespoke markup for a primitive type the library already offers is FORBIDDEN. "Closest-analog"/local-file consistency NEVER overrides the global system. Showcase: route `/dev/ui-system`. | D-14, CLAUDE.md, MEMORY |
| **C-UI-MIGRATE** | This phase ALSO fixes the existing native `<select>` in `AnimeContributionModal.tsx` (replace with `Select` + `FormField`) and aligns `ReleaseVersionBreakdown.tsx` to the primitives/tokens. | D-14 (folded todo) |
| **C-UMLAUTE** | All user-facing German strings use correct umlauts (ä ö ü Ä Ö Ü ß). ASCII replacements (ae/oe/ue/ss) are FORBIDDEN in JSX text, button labels, error messages, placeholders, aria-labels, toasts. | CLAUDE.md "Sprachqualität" |
| **C-NAMING** | Use **"Mitwirkende"** consistently (never "Beiträge") across all UI surfaces, badges, drawer title, buttons. | D-15 |
| **C-NO-DIFF** | NO "nicht dabei" / diff / "A wurde durch B ersetzt" / claim / comparison marker anywhere. Absence = the person is simply missing from the role list. A re-assignment just changes the row. | D-03, D-07, D-08, Specifics |
| **C-STAGED** | The Drawer is staged: explicit **Speichern** / **Abbrechen** only — NO auto-save on row edits. | D-06 |

## Spacing Scale

Declared values map 1:1 to the existing `--space-*` tokens (all multiples of 4). Use the token, never a raw px.

| Token | Value | CSS var | Usage in this phase |
|-------|-------|---------|---------------------|
| xs | 4px | `--space-1` | Icon-to-label gap, badge inner padding |
| sm | 8px | `--space-2` | Gap between avatar and name, compact row gaps |
| md | 16px | `--space-4` | Default gap between role rows, drawer body padding |
| lg | 24px | `--space-5` | Drawer section padding, gap above the add-row button |
| xl | 32px | `--space-6` | Major drawer section breaks |
| 2xl | 48px | `--space-7` | (rarely; reserve) |
| 3xl | 64px | `--space-8` | (rarely; reserve) |

Exceptions:
- Interactive control min-height is **44px** (`--control-height-md`) for `Button`/`Select` touch targets — established control token, not a spacing violation.
- Icon-only row buttons (`MoreHorizontal`, `Trash2`) render at 36px (`--control-height-sm`) with 18px lucide glyphs, consistent with `Drawer` close button.

## Typography

Map to `--font-sans`. Two weights only: **regular 400** and **bold 700** (the existing `ui.module.css` uses 700 for buttons/headings; 600 is NOT used in the primitive set, so we stay 400/700 to match).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Person names, hint text in the drawer body |
| Label | 14px | 700 | 1.2 | `FormField` labels, role names, badge text, button labels |
| Heading | 20px | 700 | 1.2 | Drawer title "Mitwirkende — {Release}" |
| Caption | 12px | 400 | 1.4 | Avatar fallback initials, secondary muted hints (`--text-faint`) |

Weights declared: **2** (400, 700). Sizes declared: **4** (12, 14, 16, 20).

## Color

60/30/10 split mapped to existing tokens. This is a quiet admin surface (Phase 82 directive: ruhig/admin-tauglich, keine Marketing-Optik).

| Role | Value | CSS var | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `#f6f4ef` | `--surface-canvas` | App/page background behind the cockpit |
| Secondary (30%) | `#ffffff` | `--surface-card` | Drawer panel, role-row cards, cockpit cards |
| Accent (10%) | `#5f84dd` | `--accent-primary` / `--color-primary` | RESERVED — see list below |
| Destructive | `#82122c`→`#3a0e23` gradient | `--button-danger-*` | "Entfernen" (Trash) button ONLY |

**Accent (`#5f84dd`) reserved for — explicit list, nothing else:**
1. The primary **Speichern** button in the drawer footer (`Button variant="primary"`).
2. The **"Mitwirkende"** entry action on the release row (as `subtle`/`ghost`, which tint with `--accent-primary` mix).
3. The **"Rolle / Person hinzufügen"** button (secondary/subtle accent).
4. Focus rings on interactive controls (`--focus-ring`).

Accent is NOT used for: body text, role-row borders, avatars, badges (badges use their semantic variant tokens), or decorative fills.

**Semantic / status colors (badges only, not "accent"):**
- Cockpit per-release overview badge uses `Badge variant="muted"` or `neutral` for the normal "inherited project team" state, and `variant="info"` when a release carries an explicit override set. No success/warning needed for the happy path.
- Destructive (`--button-danger-*`) is reserved strictly for the row "Entfernen" action and any confirm.

**Destructive actions in this phase:**
- **Entfernen (Mitwirkende aus diesem Release)** — `Button variant="danger" iconOnly` (Trash2). Per D-07/D-03/C-NO-DIFF, removal just drops the row; it does NOT show a "war hier nicht dabei" marker. Confirmation: because this only stages a change (C-STAGED, nothing persists until Speichern), the row is removed immediately within the staged editor with NO blocking confirm dialog. The staged-discard safety net is the **Abbrechen** button (reverts all staged changes). Persisted destructive scope (D-16 soft-delete) is a backend concern, not a UI confirm.
- If `Abbrechen` is pressed while staged changes exist: show a lightweight `Modal` confirm — heading `Änderungen verwerfen?`, body `Deine ungespeicherten Änderungen an den Mitwirkenden gehen verloren.`, primary `Verwerfen` (`variant="danger"`), secondary `Weiter bearbeiten`.

## Interaction Contract (drawer behaviour — load-bearing)

1. **Entry:** In the cockpit `tab=releases`, each release-version row carries a `Mitwirkende` action (`Button subtle/ghost sm`). Click opens the right-side `Drawer`. (D-06, D-08)
2. **On open:** The role list is **pre-filled with the inherited project team** — every contributor in their role(s) is shown immediately. There is **NO** separate "Vom Projekt geerbtes Team" block above the list. (D-07)
3. **Row anatomy:** `[role icon] {Rolle}` on the left · `[avatar] {Person}` in the middle · right-aligned actions `⋯ Rolle ändern` (`MoreHorizontal`, ghost iconOnly) + `Entfernen` (`Trash2`, danger iconOnly). (D-07)
4. **Add:** A `Rolle / Person hinzufügen` button at the list end opens an inline add row using `Select`+`FormField` for Person (from the current group only, D-12) and Role (from `role_definitions` catalog). (D-07, D-12)
5. **Re-assign A→D:** Remove the row at A, add a new row for D. The row "just changes" — NO "A war vorher hier" hint. (D-07, C-NO-DIFF)
6. **Multiple people per role allowed** (e.g. two translators) — no exclusivity enforcement in the UI. (D-09)
7. **Staged:** Edits/removals/adds only mutate local staged state. Footer `Speichern` persists the full explicit override set for exactly this release-version; `Abbrechen` discards. No auto-save. (D-06, D-10, D-11, C-STAGED)
8. **Candidate pool:** Person picker lists ONLY members of the current fansub group, even for collab releases. (D-12)
9. **Avatar:** initials fallback (first letters of name) on a tinted circle; image if available. Use `--surface-sunken` background, `--text-soft` initials, `--radius-md`.
