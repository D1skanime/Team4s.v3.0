**AniSearch HTTP fetch (per D-09, D-10):**

AniSearch is a public anime database. Fetch metadata for a given ID from:
`https://www.anisearch.com/anime/{id}` — parse the HTML response for:
- Title (h1.title or meta og:title)
- Year (production year)
- Description (meta description or .description-text)
- Type (TV, Film, OVA, etc. — map to team4s types: tv/film/ova/ona/special/bonus)
- Relations — list of related anime entries with their AniSearch IDs, titles, and relation types

If the fetch fails or returns non-200, return an enrichment error with HTTP status reason.

Use a 15-second timeout on the HTTP client. Pass `http.Client` in via the service constructor (not a package-level global).

**EnrichAndPersist method:**

```go
func (s *AnimeAniSearchEnrichmentService) EnrichAndPersist(
    ctx context.Context,
    animeID int64,
    aniSearchID string,
    lockedFields []string,
) (*AniSearchEnrichmentResult, error)
```

Steps:
1. Fetch AniSearch page for `aniSearchID`, parse draft fields and relation candidates.
2. Load current anime record from DB (SELECT id, title, title_de, title_en, year, description, type FROM anime WHERE id = $1).
3. Apply OVERRIDE mode: for each draft field, use the AniSearch value — EXCEPT skip fields present in `lockedFields`. Locked field names match JSON field names: "title", "title_de", "title_en", "year", "description", "type".
4. Resolve each relation candidate:
   a. If candidate has non-empty AniSearchID: `SELECT id FROM anime WHERE source LIKE 'anisearch:%' AND source = 'anisearch:' || $1 LIMIT 1`
   b. If no match, fallback: `SELECT id FROM anime WHERE lower(title) = lower($1) LIMIT 1`
   c. If still no match: skip, increment `SkippedRelations`.
5. Map relation type string from AniSearch to V1 label:
   - "Sequel" / "Fortsetzung" / "Continuation" → "Fortsetzung"
   - "Prequel" / "Vorgeschichte" → "Hauptgeschichte"
   - "Side Story" / "Nebenhandlung" / "Spin-off" → "Nebengeschichte"
   - "Summary" / "Zusammenfassung" → "Zusammenfassung"
   - All others → "Nebengeschichte" (safe default)
6. For resolved relations, insert into anime_relations:
   ```sql
   INSERT INTO anime_relations (source_anime_id, target_anime_id, relation_type_id)
   SELECT $1, $2, rt.id FROM relation_types rt WHERE rt.name = $3
   ON CONFLICT DO NOTHING
   ```
   Map V1 label to relation_type name:
   - "Fortsetzung" → "sequel"
   - "Hauptgeschichte" → "full-story"
   - "Nebengeschichte" → "side-story"
   - "Zusammenfassung" → "summary"
7. Return `AniSearchEnrichmentResult` with draft (post-lock-application), resolved relations with `Written: true`, skipped count.

**AniSearch HTML parsing note:** Use Go's `golang.org/x/net/html` package if available in go.mod; otherwise use `strings`/`regexp` to extract from raw HTML. Check go.mod first — if `golang.org/x/net` is present, use html.Parse. If not, use regexp on the raw body. Do NOT add new dependencies without checking go.mod.

```go
package handlers

import (
    "log"
    "net/http"

    "github.com/gin-gonic/gin"
)

type aniSearchEnrichmentRequest struct {
    AniSearchID  string   `json:"anisearch_id"`
    LockedFields []string `json:"locked_fields"`
}

type aniSearchEnrichmentResponse struct {
    AnimeID          int64                      `json:"anime_id"`
    Draft            any                        `json:"draft"`
    Relations        []aniSearchRelationSummary `json:"relations"`
    SkippedRelations int                        `json:"skipped_relations"`
}

type aniSearchRelationSummary struct {
    TargetAnimeID int64  `json:"target_anime_id"`
    RelationLabel string `json:"relation_label"`
    TargetTitle   string `json:"target_title"`
    Written       bool   `json:"written"`
}

func (h *AdminContentHandler) EnrichAnimeFromAniSearch(c *gin.Context) {
    identity, ok := h.requireAdmin(c)
    if !ok {
        return
    }

    id, err := parseAnimeID(c.Param("id"))
    if err != nil {
        log.Printf("enrich_anime_anisearch: invalid id %q (user_id=%d): %v", c.Param("id"), identity.UserID, err)
        badRequest(c, "ungueltige anime id")
        return
    }

    var req aniSearchEnrichmentRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        badRequest(c, "ungueltiger request body")
        return
    }

    if req.AniSearchID == "" {
        badRequest(c, "anisearch_id ist erforderlich")
        return
    }

    result, err := h.aniSearchEnrichment.EnrichAndPersist(c.Request.Context(), id, req.AniSearchID, req.LockedFields)
    if err != nil {
        log.Printf("enrich_anime_anisearch: service error (user_id=%d, anime_id=%d): %v", identity.UserID, id, err)
        c.JSON(http.StatusBadGateway, gin.H{"error": gin.H{"message": "AniSearch-Anfrage fehlgeschlagen"}})
        return
    }

    // Build response — map service types to response types
    relations := make([]aniSearchRelationSummary, 0, len(result.Relations))
    for _, r := range result.Relations {
        relations = append(relations, aniSearchRelationSummary{
            TargetAnimeID: r.TargetAnimeID,
            RelationLabel: r.RelationLabel,
            TargetTitle:   r.TargetTitle,
            Written:       r.Written,
        })
    }

    c.JSON(http.StatusOK, gin.H{
        "data": gin.H{
            "anime_id":          result.AnimeID,
            "draft":             result.Draft,
            "relations":         relations,
            "skipped_relations": result.SkippedRelations,
        },
    })
}
```

| Boundary | Description |
|----------|-------------|
| Admin client → enrichment endpoint | Authenticated admin sends anisearch_id and locked_fields; anisearch_id is used as a URL path segment to an external HTTP call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Spoofing | EnrichAnimeFromAniSearch handler | mitigate | requireAdmin() check enforced before any logic executes |
| T-10-02 | Tampering | anisearch_id used in URL | mitigate | Validate anisearch_id is numeric-only (strip non-digit chars or reject on non-numeric) before constructing the outbound URL to prevent path traversal |
| T-10-03 | Denial of Service | Outbound AniSearch HTTP fetch | mitigate | 15-second http.Client timeout; no retry loop; one request per handler call |
| T-10-04 | Information Disclosure | AniSearch HTML parse errors logged | accept | Logs are internal-only; no HTML content is echoed to the client |
| T-10-05 | Elevation of Privilege | locked_fields bypass | accept | locked_fields is admin-controlled; if admin wants to unlock a field, that is intentional |
| T-10-06 | Tampering | relation INSERT with user-supplied target_anime_id | mitigate | target_anime_id is resolved by internal DB lookup from title/source, never taken directly from AniSearch HTML |
</threat_model>
