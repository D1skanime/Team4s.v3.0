<threat_model>
## Trust Boundaries

| Boundary | Beschreibung |
|----------|-------------|
| Client → /unified-members | Nicht-Gruppenmitglieder dürfen die Personenliste nicht sehen |
| Client → Contribution-Mutation | Nur Mitglieder der Gruppe dürfen Contributions anlegen (Cross-Group-Guard) |
| member_id in Request | Ein böswilliger Client könnte fremde member_id-Werte einschleusen |
| Client → /default-crew PUT | Nur Gruppenleader mit MembersManage dürfen Stamm-Crew-Einträge anlegen |
| Client → /default-crew/apply | apply muss idempotent und auf eigene Gruppe beschränkt sein |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-02-01 | Elevation of Privilege | GET /unified-members | mitigate | CanForFansubGroup(MembersView) Permission-Guard vor Handler-Logik (ASVS V4.1) |
| T-82-02-02 | Spoofing | animeContributionCreateRequest.MemberID | mitigate | MemberBelongsToFansub UNION-Check validiert member_id gegen hist + app für die gesendete fansubID |
| T-82-02-03 | Tampering | Contribution-Mutation mit fremder member_id | mitigate | Cross-Group-Guard: MemberBelongsToFansub gibt false wenn member nicht zu Gruppe gehört → 422 zurück |
| T-82-02-04 | Information Disclosure | ListUnifiedByFansub gibt App-Member inkl. has_app_account zurück | mitigate | Endpoint nur für authentifizierte Gruppenleader (authMiddleware + MembersView) |
| T-82-02-05 | Elevation of Privilege | PUT /default-crew — fremde member_id einschleusen | mitigate | Cross-Group-Guard: PutDefaultCrew ruft MemberBelongsToFansub auf; bei false → 422 |
| T-82-02-06 | Elevation of Privilege | POST /default-crew/apply — Contributions für fremde Gruppe anlegen | mitigate | fansubID kommt aus URL-Param, der gegen authMiddleware-Session geprüft wird; CanForFansubGroup(MembersManage) blockt Nicht-Leader |
| T-82-02-07 | Tampering | DELETE /default-crew/:memberId/:roleCode — fremden Eintrag löschen | mitigate | WHERE-Klausel enthält fansub_group_id aus validiertem URL-Param; Permission-Guard MembersManage |
| T-82-02-SC | Tampering | npm/pip/cargo installs | accept | keine Package-Installs in diesem Plan |
