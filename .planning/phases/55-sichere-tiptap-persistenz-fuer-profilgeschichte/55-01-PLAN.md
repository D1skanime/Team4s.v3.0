## Trust Boundaries

| Boundary | Beschreibung |
|----------|--------------|
| Browser -> Profile API | Client sendet TipTap JSON, aber keine HTML-Wahrheit |
| Profile API -> Renderer | Backend rendert und sanitiziert HTML serverseitig |
| Accountdaten -> Profilgeschichte | Keycloak-/Accountfelder bleiben read-only und getrennt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-01 | Tampering | TipTap JSON input | mitigate | `TipTapService.ValidateJSON` vor Persistenz; unbekannte Nodes/Marks 400 |
| T-55-02 | XSS | member_story_html | mitigate | HTML ausschließlich serverseitig aus JSON erzeugen und sanitizen; `RichTextRenderer` konsumiert nur sanitisiertes HTML |
| T-55-03 | Information Disclosure | Profile aggregate | mitigate | Keine Account-/Gruppenrollen-Daten in Story-Felder mischen; bestehende Profil-Capabilities bleiben getrennt |
| T-55-04 | Data Loss | Plain-text conversion | mitigate | TipTap JSON als Quelle der Wahrheit; Plain Text nur abgeleitet |
</threat_model>
