<ui_contract>
No UI-SPEC file exists for Phase 18, so implement this essential contract directly:
- Route: `/admin/anime/[id]/episodes/import`.
- Entry: add a secondary action on `/admin/anime/[id]/episodes` labeled `Import & Mapping` or equivalent German copy, next to existing overview actions.
- Sections:
  1. Anime context: title, anime ID, AniSearch source ID when present, Jellyfin folder/path/series ID when present.
  2. Source controls: AniSearch ID input seeded from `source='anisearch:{id}'`; Jellyfin series/folder context read-only by default; optional season offset numeric input used only for suggestions.
  3. Preview summary: counts for canonical episodes, media candidates, suggested mappings, conflicts, skipped/unmapped canonical episodes, and unmapped media files.
  4. Canonical episode list: episode number, AniSearch title, existing local episode marker.
  5. Jellyfin media candidates: file name/path, `SxxExx` evidence, media item ID, stream URL availability, quality.
  6. Mapping builder rows: one row per media candidate with editable comma-separated target episode numbers, status pill (`suggested`, `confirmed`, `conflict`, `skipped`), confirm/skip controls, and warning copy when a media row maps to multiple episodes.
  7. Apply result: counts from backend apply response: episodes created/existing, versions created/updated, mappings applied, skipped, conflicts.
- UX rules:
  - Preview never applies automatically.
  - Apply button disabled while any row remains `suggested` or `conflict`; skipped rows are allowed.
  - Multi-target input such as `9,10` is accepted and normalized to `[9,10]`.
  - Unmapped canonical episodes and unmapped files remain visible instead of disappearing.
  - Do not redesign public playback or the broader Anime Edit surface.
  - Reuse existing admin/create visual language; avoid a landing page or marketing copy.
</ui_contract>
</context>
