**Step A — Extend `useReleaseVersionMedia.ts` to add upload state management:**

Add to the `UseReleaseVersionMediaResult` interface:
```typescript
uploadItems: UploadQueueItem[]        // per-file upload state
startUpload: (category: ReleaseVersionMediaCategory, files: File[], defaultCaption?: string, isPreviewCandidate?: boolean) => Promise<void>
retryUpload: (fileIndex: number) => Promise<void>
clearUploadQueue: () => void
```

Where `UploadQueueItem` is:
```typescript
export interface UploadQueueItem {
  file: File
  status: 'idle' | 'uploading' | 'processing' | 'ready' | 'failed'
  progress: number   // 0–100
  errorMessage: string | null
  resultId: number | null   // release_version_media_id when ready
}
```

`startUpload` calls `uploadReleaseVersionMedia` with `onProgress` updating per-file progress. On completion, for each result:
- `status=ready` → mark queue item as ready, then call `reload()` to refresh the confirmed gallery (D-11: no optimistic insert)
- `status=failed` or `status=processing` → mark queue item as failed with errorMessage from error_code
