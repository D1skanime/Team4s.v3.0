## Goal

Zwei Repository-Dateien für alle 4 Note-Typen erstellen. `fansub_notes_repository.go` enthält CRUD für die drei fansub-seitigen Note-Typen. `release_version_notes_repository.go` enthält Bulk-Upsert für Release-Version-Notizen sowie eine Hilfsmethode zum Laden der beteiligten Member+Rollen.

- `40-CONTEXT.md` — Bulk-Save-Entscheidung, Eindeutigkeitsregel für release_version_notes
- `backend/internal/repository/release_version_media_repository.go` — kanonisches Muster (soft delete, pgxpool, error handling)

**ReleaseVersionNotesRepository** mit Konstruktor `NewReleaseVersionNotesRepository(db *pgxpool.Pool) *ReleaseVersionNotesRepository`.

Methoden:

- `ListReleaseVersionNotes(ctx, releaseVersionID int64) ([]ReleaseVersionNote, error)` — WHERE deleted_at IS NULL, ORDER BY sort_order, member_id, role_id

- `GetMemberRolesForVersion(ctx, releaseVersionID int64) ([]MemberRoleForVersion, error)` — kanonischer Join-Pfad aus RESEARCH.md:
  ```sql
  SELECT rmr.member_id, m.nickname AS member_name, rmr.role_id, cr.name AS role_name, cr.label AS role_label
  FROM release_member_roles rmr
  JOIN members m ON m.id = rmr.member_id
  JOIN contributor_roles cr ON cr.id = rmr.role_id
  JOIN release_versions rv ON rv.release_id = rmr.release_id
  WHERE rv.id = $1
  ORDER BY cr.name, m.nickname
  ```

- `BulkUpsertReleaseVersionNotes(ctx, releaseVersionID int64, userID int64, notes []BulkNoteInput) ([]ReleaseVersionNote, error)` — User-Entscheidung 3: Bulk-Save. Für jedes Element in `notes`:
  - Wenn `id == 0`: INSERT mit RETURNING id
  - Wenn `id > 0`: UPDATE WHERE id = $x AND release_version_id = $y AND deleted_at IS NULL
  - UNIQUE-Verletzung auf (release_version_id, member_id, role_id): als ErrConflict zurückgeben
  - Transaktionsartig: alle in einer DB-Transaktion ausführen
  - Gibt die vollständige Liste aller aktiven Notes nach dem Upsert zurück

- `DeleteReleaseVersionNote(ctx, noteID int64, releaseVersionID int64, userID int64) error` — soft delete

**BulkNoteInput** Struct:
