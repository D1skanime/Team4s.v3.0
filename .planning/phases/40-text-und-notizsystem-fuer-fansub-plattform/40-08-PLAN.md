## Goal

Neuen Tab "Notizen" im Fansub-Gruppen-Editor (`/admin/fansubs/[id]/edit`) hinzufügen. Der Tab zeigt zwei Abschnitte: Gruppennotizen (`fansub_group_notes`) und Mitgliedergeschichten (`member_group_stories`). Jeder Abschnitt listet bestehende Einträge und ermöglicht Erstellen, Bearbeiten und Löschen. Markdown-Editor mit Toolbar und `<pre>`-Vorschau wird wiederverwendet (bestehend im Fansub-Editor).

## Context

- `frontend/src/app/admin/fansubs/[id]/edit/page.tsx` — bestehende Tab-Struktur, Markdown-Toolbar-Komponente
- `RESEARCH.md` — Frontend Patterns: Split-View, Toolbar, MARKDOWN_SOFT_LIMIT
- `40-CONTEXT.md` — UI-Beschreibungen, Sichtbarkeitsregel, Status-Werte
- Plan 40-07 SUMMARY — TypeScript-Typen und API-Funktionen (listFansubGroupNotes etc.)

## Tasks

### Task 1: NotesTab.tsx erstellen

Erstelle `frontend/src/app/admin/fansubs/[id]/edit/NotesTab.tsx`.

**Props:**
```typescript
interface NotesTabProps {
  fansubId: number;
}
```

**Inhaltsstruktur** des Tabs:

Abschnitt 1: "Gruppennotizen"
- Überschrift: "Gruppennotizen"
- Beschreibung: "Offizielle Texte über die Gruppe: Geschichte, Philosophie, Stil, Abschlüsse."
- Liste bestehender fansub_group_notes (geladen beim Mount via `listFansubGroupNotes`)
- Jede Note: Titelfeld, Markdown-Editor mit Toolbar und `<pre>`-Vorschau (bestehende Toolbar wiederverwenden), Visibility-Select (`öffentlich`/`intern`), Status-Select (`Entwurf`/`Veröffentlicht`/`Archiviert`/`Gelöscht`), SortOrder-Feld, Speichern-Button, Löschen-Button
- "Neue Gruppennotiz hinzufügen"-Button fügt leeres Formular hinzu

Abschnitt 2: "Mitgliedergeschichten"
- Überschrift: "Mitgliedergeschichten"
- Beschreibung: "Persönliche Erinnerungen einzelner Mitglieder in dieser Gruppe."
- Liste bestehender member_group_stories (geladen beim Mount via `listMemberGroupStories`)
- Jede Story: Mitglied-ID-Feld (oder Mitglied-Select wenn Members-Liste verfügbar), optionales Rollen-Select (`contributor_roles`), Titelfeld, Markdown-Editor, Visibility, Status, SortOrder, Speichern/Löschen
- "Neue Geschichte hinzufügen"-Button

**State-Management:** Lokaler React-State mit `useState` und `useEffect` für den initialen Load. Kein globaler State nötig.

**Fehlerbehandlung:** Fehler beim API-Call im State speichern und als Text unterhalb der Aktion anzeigen ("Fehler beim Speichern: ...").

**Ladezustand:** `isLoading`-State, während Laden: "Wird geladen..." anzeigen.

**Markdown-Editor-Wiederverwendung:** Die bestehende Toolbar-Komponente aus `page.tsx` extrahieren oder direkt referenzieren. Wenn sie als lokale Funktion/Komponente in `page.tsx` definiert ist, in eine eigene Datei auslagern (`MarkdownToolbar.tsx`) und von beiden Dateien importieren. Entscheide nach Lesen von `page.tsx`.

**Sprachqualität:** Alle Buttons, Labels, Placeholder auf Deutsch mit korrekten Umlauten:
- "Gruppennotizen", "Mitgliedergeschichten", "Titel", "Inhalt"
- "Öffentlich" / "Intern", "Entwurf" / "Veröffentlicht" / "Archiviert" / "Gelöscht"
- "Speichern", "Löschen", "Neue Gruppennotiz hinzufügen", "Neue Geschichte hinzufügen"
- "Wird geladen...", "Fehler beim Speichern"

### Task 2: Tab-Integration in page.tsx

Öffne `frontend/src/app/admin/fansubs/[id]/edit/page.tsx`.

1. Füge `'notes'` zum Tab-Array hinzu: nach dem letzten bestehenden Tab
2. Tab-Label: `"Notizen"`
3. Render-Block für `activeTab === 'notes'`: `<NotesTab fansubId={fansubId} />`
4. Import von `NotesTab` hinzufügen

**Wichtig:** Keine anderen Teile von `page.tsx` ändern — nur Tab-Array, Render-Condition und Import.
