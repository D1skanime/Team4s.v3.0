## Goal

Neuen Tab "Notizen / Beiträge" im Release-Version-Editor hinzufügen. Der Tab zeigt pro beteiligtem Member+Rolle ein Textfeld mit rollenspezifischen Hilfetexten und Placeholdern (aus CONTEXT.md). Speichern erfolgt per Bulk-Save (ein Request für alle Felder gleichzeitig, User-Entscheidung 3).

- `40-CONTEXT.md` — vollständige Hilfetexte pro Rolle (Tabelle mit Label, Hilfetext, Placeholder)
- Plan 40-07 SUMMARY — TypeScript-Typen und API-Funktionen

**Allgemeiner Hinweistext** (oben im Tab anzeigen):
> "Diese Notizen beschreiben die konkrete Release-Version. Schreibe kurz, was du in deiner Rolle gemacht hast oder was an dieser Ausgabe besonders war. 2–5 Sätze reichen."

**Lade-Logik beim Mount:**

1. `getMemberRolesForVersion(versionId)` → Liste der beteiligten Member+Rollen
2. `listReleaseVersionNotes(versionId)` → bestehende Notes

Beide Calls parallel via `Promise.all`.

**State:**

```typescript
// Pro MemberRole ein Formularfeld-State
type NoteFormState = {
  id: number;  // 0 = noch nicht in DB
  bodyMarkdown: string;
  title: string;
  visibility: 'public' | 'internal';
  status: 'draft' | 'published' | 'archived' | 'deleted';
  sortOrder: number;
  isDirty: boolean;
};
// Key: `${memberId}-${roleId}`
const [noteStates, setNoteStates] = useState<Record<string, NoteFormState>>({});
```

**Darstellung:**

- Pro Member eine Karte/Accordion
- Innerhalb der Karte: pro Rolle ein Textbereich
- Label der Rolle anzeigen (aus `ROLE_HELP_TEXTS[roleName].label` oder `memberRole.roleLabel`)
- Hilfetext als kleiner Text unter dem Label
- Textarea mit Placeholder aus `ROLE_HELP_TEXTS`
- Optional: Title-Feld (ausklappbar/klein)
- Visibility-Select: `öffentlich` / `intern`
- Status-Select: `Entwurf` / `Veröffentlicht` / `Archiviert` / `Gelöscht`

**Bulk-Save-Button:**

- Label: "Alle Notizen speichern"
- Sammelt alle aktuellen Formularzustände
- Baut `BulkUpsertReleaseVersionNotesRequest` mit `notes: [...]` auf
- Leere `bodyMarkdown`-Felder: NICHT im Bulk-Request mitsenden wenn keine ID vorhanden (nicht neu erstellen wenn leer)
- Bestehende Notes (id > 0): immer mitsenden auch wenn leer (ermöglicht Löschen des Inhalts)
- `bulkUpsertReleaseVersionNotes(versionId, request)` aufrufen
- Erfolg: kurze Erfolgsmeldung, neue IDs in State übernehmen
- 409: "Fehler: Für ein Mitglied und eine Rolle existiert bereits eine Notiz."
- Sonstige Fehler: "Fehler beim Speichern. Bitte erneut versuchen."

**Sonderfall: leere Member-Rollen-Liste:**
Wenn `getMemberRolesForVersion` leere Liste zurückgibt (historische Daten ohne release_member_roles):
Hinweistext: "Für diese Release-Version sind keine Mitglieder und Rollen zugeordnet. Notizen können erst hinzugefügt werden, wenn Mitglieder im Release zugeordnet sind."

**Zeichenlimit-Hinweis** (aus CONTEXT.md: max. 2000–4000 Zeichen):
Unter jedem Textarea: Zeichenzähler, ab 2000 Zeichen gelber Hinweis "Empfohlene Länge überschritten (2000 Zeichen)".
