## Goal

HTTP-Handler und Routing für die drei fansub-seitigen Note-Typen erstellen. Die Datei `admin_content_fansub_notes.go` hängt sich an den bestehenden `AdminContentHandler` (Receiver-Methoden). Routing erfolgt in `admin_routes.go`. `main.go` wird minimal erweitert um `FansubNotesRepository` und `MarkdownService` zu instanziieren und zu verdrahten.

**Modularitäts-Constraint:** Die neue Datei darf max. 450 Zeilen haben (CLAUDE.md). Bei Überschreitung aufteilen (z.B. separate Datei pro Note-Typ).

## Tasks

### Task 1: Handler-Datei admin_content_fansub_notes.go

Erstelle `backend/internal/handlers/admin_content_fansub_notes.go`.

Dieses File ist eine Erweiterung des `AdminContentHandler` (Receiver-Methoden auf `*AdminContentHandler`). Für den Service-Zugriff muss `AdminContentHandler` um folgende Felder erweitert werden — diese Felder werden in `main.go` befüllt:

- `fansubNotesRepo *repository.FansubNotesRepository`
- `markdownSvc *services.MarkdownService`

Die Erweiterung des Handler-Structs erfolgt in `admin_content_handler.go` oder einer neuen Erweiterungs-Datei — je nach wie der bestehende Handler organisiert ist. Prüfe erst `admin_content_handler.go`, dann entscheide.

**Auth-Hilfsfunktion** (im gleichen File):

```go
// requireFansubGroupNoteWriteAccess kapselt die Berechtigungsprüfung für
// fansub_group_notes und member_group_stories. MVP: Admin-only.
// Eigene Funktion ermöglicht spätere Erweiterung ohne Änderung an Call-Sites.
func (h *AdminContentHandler) requireFansubGroupNoteWriteAccess(c *gin.Context) (middleware.AuthIdentity, bool) {
    return h.requireAdmin(c)
}
```

**Request-Structs** (im gleichen File):

```go
type createFansubGroupNoteRequest struct {
    Title      string `json:"title"`
    BodyMarkdown string `json:"body_markdown" binding:"required"`
    Visibility string `json:"visibility" binding:"required,oneof=public internal"`
    Status     string `json:"status" binding:"required,oneof=draft published archived deleted"`
    SortOrder  int    `json:"sort_order"`
}

type updateFansubGroupNoteRequest struct {
    Title      *string `json:"title"`
    BodyMarkdown *string `json:"body_markdown"`
    Visibility *string `json:"visibility"`
    Status     *string `json:"status"`
    SortOrder  *int    `json:"sort_order"`
}
// Analoge Structs für MemberGroupStory und AnimeFansubProjectNote
```

**Handler-Methoden** für `fansub_group_notes`:

- `ListFansubGroupNotes(c *gin.Context)` — GET, liest `:id` als fansubGroupID, ruft `h.fansubNotesRepo.ListFansubGroupNotes`, gibt JSON zurück
- `CreateFansubGroupNote(c *gin.Context)` — POST, Auth-Check, BodyMarkdown → BodyHTML via `h.markdownSvc.RenderMarkdown`, dann `h.fansubNotesRepo.CreateFansubGroupNote`
- `UpdateFansubGroupNote(c *gin.Context)` — PATCH, Auth-Check, `:noteId` als noteID, re-render HTML wenn BodyMarkdown übergeben
- `DeleteFansubGroupNote(c *gin.Context)` — DELETE, Auth-Check

**Handler-Methoden** für `member_group_stories` (analog, zusätzlich `member_id` und `role_id` im Body):

- `ListMemberGroupStories`, `CreateMemberGroupStory`, `UpdateMemberGroupStory`, `DeleteMemberGroupStory`

**Handler-Methoden** für `anime_fansub_project_notes`:

- `GetAnimeFansubProjectNote(c *gin.Context)` — GET, liest `:id` (fansubGroupID) und `:animeId`
- `UpsertAnimeFansubProjectNote(c *gin.Context)` — POST/PUT, Auth-Check, Markdown-Rendering, Upsert
- `DeleteAnimeFansubProjectNote(c *gin.Context)` — DELETE, Auth-Check

Response-Format: `gin.H{"data": note}` für Einzelobjekte, `gin.H{"data": notes}` für Listen (bestehende Konvention aus RESEARCH.md).

Fehlerbehandlung: `repository.ErrNotFound` → 404, `repository.ErrConflict` → 409, sonstige → 500.

### Task 2: Routing in admin_routes.go

In `backend/cmd/server/admin_routes.go` die neuen Routen hinzufügen. Suche den bestehenden Fansub-Routing-Block (wo `/admin/fansubs/:id/...`-Routen registriert sind) und ergänze:

```go
// fansub_group_notes
fansub.GET("/:id/notes", h.ListFansubGroupNotes)
fansub.POST("/:id/notes", h.CreateFansubGroupNote)
fansub.PATCH("/:id/notes/:noteId", h.UpdateFansubGroupNote)
fansub.DELETE("/:id/notes/:noteId", h.DeleteFansubGroupNote)

// member_group_stories
fansub.GET("/:id/member-stories", h.ListMemberGroupStories)
fansub.POST("/:id/member-stories", h.CreateMemberGroupStory)
fansub.PATCH("/:id/member-stories/:storyId", h.UpdateMemberGroupStory)
fansub.DELETE("/:id/member-stories/:storyId", h.DeleteMemberGroupStory)

// anime_fansub_project_notes
fansub.GET("/:id/anime/:animeId/notes", h.GetAnimeFansubProjectNote)
fansub.PUT("/:id/anime/:animeId/notes", h.UpsertAnimeFansubProjectNote)
fansub.DELETE("/:id/anime/:animeId/notes/:noteId", h.DeleteAnimeFansubProjectNote)
```

Routing-Variablen-Namen müssen mit den Handler-Aufrufen von `c.Param(...)` übereinstimmen.
