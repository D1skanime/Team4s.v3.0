## Goal

Eine UI-Sektion für Anime-Projekttexte (`anime_fansub_project_notes`) im Fansub-Editor. Diese Sektion zeigt für jede Anime-Zuordnung der Gruppe ein Textfeld mit Schreibimpulsen aus CONTEXT.md. Sie kann entweder als eigener Tab oder als Section innerhalb des bestehenden "Content"-Tabs eingebaut werden — entscheide nach Lesen von `page.tsx`, was weniger invasiv ist.

## Context

- `frontend/src/app/admin/fansubs/[id]/edit/page.tsx` — bestehende Tab-Struktur, Anime-Zuordnungen (prüfe ob die Gruppe ihre Anime-Zuordnungen bereits lädt)
- `RESEARCH.md` — Frontend Patterns
- `40-CONTEXT.md` — Schreibimpulse, Zielmodell, MVP-Haupttext pro Gruppe/Anime
- Plan 40-07 SUMMARY — TypeScript-Typen und API-Funktionen (getAnimeFansubProjectNote etc.)

## Tasks

### Task 1: AnimeProjectNotesSection.tsx erstellen

Erstelle `frontend/src/app/admin/fansubs/[id]/edit/AnimeProjectNotesSection.tsx`.

**Props:**
```typescript
interface AnimeProjectNotesSectionProps {
  fansubId: number;
  // Die Anime-Liste kommt von der übergeordneten Seite
  // (oder wird hier selbst geladen falls keine Props verfügbar)
  animes?: Array<{ id: number; title: string }>;
}
```

**Verhalten:**

1. **Anime-Zuordnungen laden:** Prüfe in `page.tsx` ob die bestehende Fansub-Editor-Seite bereits die Anime-Zuordnungen lädt. Wenn ja: als Props übergeben. Wenn nein: eigenen API-Call machen (Endpunkt prüfen: z.B. `GET /admin/fansubs/:id/anime` — oder bestehende Struktur recherchieren).

2. **Pro Anime ein Accordion/Card:**
   - Anime-Titel als Überschrift
   - Textfeld mit Markdown-Editor
   - Placeholder-Text (Schreibimpulse aus CONTEXT.md):
     ```
     Beschreibe hier das Fansubprojekt dieser Gruppe zu diesem Anime.
     Mögliche Fragen als Hilfe: Wie war dieses Fansubprojekt? Warum hat die Gruppe diesen Anime gemacht?
     Was war besonders? Wie lief die Arbeit? Gab es Coop? Gab es Re-Releases? Gab es Probleme/Abbrüche?
     Welche Rollen waren besonders wichtig? Schöne/schwierige Erinnerungen?
     ```
   - Titelfeld (optional)
   - Visibility-Select
   - Status-Select
   - Speichern-Button

3. **Load:** Beim Mount für jede Anime-ID: `getAnimeFansubProjectNote(fansubId, animeId)`. Falls 404/null: leeres Formular anzeigen.

4. **Save:** `upsertAnimeFansubProjectNote(fansubId, animeId, data)`.

5. **Leere Gruppen:** Wenn keine Anime-Zuordnungen vorhanden: "Diese Gruppe hat noch keine Anime-Zuordnungen." anzeigen.

**Sprachqualität:** "Anime-Projekttexte", "Projekttext für [Anime-Titel]", "Speichern", "Wird geladen...", alle korrekte Umlaute.

### Task 2: Integration in page.tsx

Entscheide nach Lesen von `page.tsx`:

**Option A (empfohlen wenn "content"-Tab wenig gefüllt):** Section innerhalb des bestehenden `'content'`-Tabs einbauen, nach bestehenden Inhalten.

**Option B:** Neuer Tab `'anime-projekte'` mit Label "Anime-Projekte".

Für gewählte Option: Import von `AnimeProjectNotesSection` und Einbau am passenden Ort. Keine anderen Teile von `page.tsx` ändern.
