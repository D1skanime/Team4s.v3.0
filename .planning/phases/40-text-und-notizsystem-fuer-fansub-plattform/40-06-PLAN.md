## Goal

HTTP-Handler und Routing für `release_version_notes` erstellen. Der Kernendpunkt ist `POST /admin/release-versions/:versionId/notes` mit Array-Payload (Bulk-Save, User-Entscheidung 3). Eine Hilfsmethode lädt die beteiligten Member+Rollen für eine Release-Version, damit die Frontend-UI die richtigen Felder anzeigen kann.

## Tasks

### Task 1: Handler-Datei admin_content_release_version_notes.go

Erstelle `backend/internal/handlers/admin_content_release_version_notes.go`.

Receiver-Methoden auf `*AdminContentHandler` (wie Plan 40-05).

**Auth-Hilfsfunktion:**

```go
// requireReleaseVersionNoteWriteAccess kapselt die Berechtigungsprüfung für
// release_version_notes. MVP: Admin-only.
func (h *AdminContentHandler) requireReleaseVersionNoteWriteAccess(c *gin.Context) (middleware.AuthIdentity, bool) {
    return h.requireAdmin(c)
}
```

**Request-Structs:**

```go
// Einzelnes Note-Element im Bulk-Save-Array
type bulkNoteItemRequest struct {
    ID           int64   `json:"id"`           // 0 = neu erstellen
    MemberID     int64   `json:"member_id" binding:"required"`
    RoleID       int64   `json:"role_id" binding:"required"`
    Title        *string `json:"title"`
    BodyMarkdown string  `json:"body_markdown"`
    Visibility   string  `json:"visibility" binding:"required,oneof=public internal"`
    Status       string  `json:"status" binding:"required,oneof=draft published archived deleted"`
    SortOrder    int     `json:"sort_order"`
}

// Bulk-Save-Request-Body
type bulkUpsertReleaseVersionNotesRequest struct {
    Notes []bulkNoteItemRequest `json:"notes" binding:"required"`
}
```

**Handler-Methoden:**

- `ListReleaseVersionNotes(c *gin.Context)` — GET `:versionId`, gibt `{"data": [...]}` zurück

- `GetMemberRolesForVersion(c *gin.Context)` — GET `:versionId/member-roles`, gibt `{"data": [{member_id, member_name, role_id, role_name, role_label}, ...]}` zurück. Wird vom Frontend genutzt um die richtigen Rollenfelder anzuzeigen.

- `BulkUpsertReleaseVersionNotes(c *gin.Context)` — POST `:versionId/notes`. Flow:
  1. Auth-Check mit `requireReleaseVersionNoteWriteAccess`
  2. Parse `bulkUpsertReleaseVersionNotesRequest`
  3. Für jede Note: `BodyHTML, err = h.markdownSvc.RenderMarkdown(note.BodyMarkdown)` — bei Fehler 500
  4. Baue `[]repository.BulkNoteInput` auf
  5. Rufe `h.releaseVersionNotesRepo.BulkUpsertReleaseVersionNotes(ctx, versionID, userID, inputs)`
  6. Bei `ErrConflict`: 409 mit Fehlermeldung "Für dieses Mitglied und diese Rolle existiert bereits eine Notiz"
  7. Erfolg: 200 mit `{"data": updatedNotes}`

- `DeleteReleaseVersionNote(c *gin.Context)` — DELETE `:versionId/notes/:noteId`, Auth-Check, soft delete

**Maximale Dateigröße:** Ziel < 250 Zeilen.
