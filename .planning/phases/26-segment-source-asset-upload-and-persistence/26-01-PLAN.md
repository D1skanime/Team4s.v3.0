<task type="auto" tdd="true">
  <name>Task 4: Eigene Segment-Asset-Routen und Handler</name>
  <files>
    backend/internal/handlers/admin_content_anime_theme_segments.go,
    backend/cmd/server/admin_routes.go
  </files>
  <action>
Implementiere:
- `POST /api/v1/admin/anime/:id/segments/:segmentId/asset`
- `DELETE /api/v1/admin/anime/:id/segments/:segmentId/asset`

Upload-Flow:
1. Segment fuer `animeID + segmentID` laden
2. Falls Segment bereits `source_ref != ""`: bestehende Datei zuerst aufraumen (wie Delete-Flow Schritt 2-3), bevor neue Datei gespeichert wird — verhindert Datei-Leichen auf Disk
3. `groupID`, `version`, `theme_type_name` aus Segment ableiten
4. `SaveSegmentAsset(...)`
5. `CreateMediaAsset(...)`
6. Segment patchen: `source_type = 'release_asset'`, `source_ref = relPath`, `source_label = original filename`
7. aktualisiertes Segment zurueckgeben

Delete-Flow:
1. `ClearSegmentAsset(...)` → gibt `previousRef` (relativer Pfad) zurueck
2. Datei loeschen: `os.Remove(filepath.Join(h.mediaStorageDir, previousRef))` — kein Fehler wenn Datei fehlt
3. `media_assets`-Zeile loeschen: `GetMediaAssetByFilename(filepath.Join(h.mediaStorageDir, previousRef))` → `DeleteMediaAsset` — kein Fehler wenn nicht gefunden
4. `204 No Content`

Wichtig:
- nicht ueber die alten `theme-assets`-Routen gehen
- keine Kopplung an Fansub-Edit oder Release-Theme-Asset-Tabellen
  </action>
  <done>Es gibt einen echten Segment-Asset-Upload/Delete-Pfad</done>
</task>
